# 3.0 小版本发布计划 — Jira Core 完整版

> **更新时间**：2026-06-03
>
> **发布时间线**：2026-12-01 ~ 2027-03-27
>
> **路线原则**：V3.0 收口 Jira Core 的关键缺口，不把 IM、OKR、Portfolio、审批流等企业协作插件作为 Core 发布阻塞项。
>
> **主路线图**：见 [ROADMAP-CORE-FIRST.zh-CN.md](../ROADMAP-CORE-FIRST.zh-CN.md)。

---

## 3.0 核心目标

V3.0 的目标是让 LEST 具备一个成熟 Jira Core 应有的完整业务能力：任务关联、附件、工时、自动化、敏捷报表、批量操作、搜索过滤、导入导出。

### 3.0 范围

| 模块 | 交付 |
|------|------|
| Issue Link | 关联类型、双向关系、阻塞校验 |
| Attachment | 上传、预览、版本、权限、存储策略 |
| Time Tracking | 预估、剩余、已记录、Worklog、汇总 |
| Automation | 规则、触发器、条件、动作、日志、模板 |
| Agile Reports | Burndown、CFD、Velocity、Control Chart、Epic Progress |
| Bulk Operations | 批量编辑、移动、删除、流转、审计 |
| Search & Filter | 完整 JQL、保存过滤器、订阅、导出 |
| Import | Jira/禅道导入、字段映射、迁移报告 |

### 3.0 不做

| 功能 | 后续版本 |
|------|----------|
| 企业微信/钉钉/飞书深度集成 | V4.0 官方插件池 |
| OKR / Portfolio / 审批流 | V5.0 企业版或 V4.0 商业插件 |
| LDAP / SSO / 多租户 | V5.0 企业版 |
| 插件市场 | V4.0 |

## 小版本规划

### 3.1 — Issue Link 与 Attachment（2026-12-12）

> **目标**：补齐任务关联和附件体系。

| 任务 ID | 交付项 | 说明 |
|---------|--------|------|
| V30-LINK-001 | 关联类型模型 | blocks、duplicates、relates、causes、clones 等 |
| V30-LINK-002 | 任务关联 API | 创建、删除、列表、反向关系展示 |
| V30-LINK-003 | 阻塞校验 | blocked by 未完成时禁止完成任务 |
| V30-LINK-004 | 关联 UI | 任务详情中按类型分组展示 |
| V30-ATT-001 | 附件模型 | 元数据、版本、上传人、权限、软删除 |
| V30-ATT-002 | 附件 API | 上传、下载、预览、删除、恢复 |
| V30-ATT-003 | 存储策略 | 本地/MinIO，预留 STORAGE_PROVIDER |
| V30-TEST | 测试 | 关联循环、附件大小、权限越界 |

**交付物**：任务关联与附件管理完整可用。

---

### 3.2 — Time Tracking 与回收站（2027-01-02）

> **目标**：实现 Jira 工时追踪和可恢复删除。

| 任务 ID | 交付项 | 说明 |
|---------|--------|------|
| V31-TIME-001 | 工时字段 | original estimate、remaining、time spent |
| V31-TIME-002 | Worklog | 新增、编辑、删除、按用户/任务/项目汇总 |
| V31-TIME-003 | 自动重算 | 登记工时后自动更新 remaining |
| V31-TIME-004 | 工时 UI | 任务详情、列表、看板卡片展示 |
| V31-TRASH-001 | 回收站模型 | deleted_at、deleted_by、restore_token |
| V31-TRASH-002 | 恢复/彻底删除 | 项目级和全局回收站 |
| V31-TRASH-003 | 自动清理 | 超期任务定时清理 |
| V31-TEST | 测试 | 工时汇总、恢复链路、级联删除 |

**交付物**：Time Tracking 和回收站完整可用。

### 3.3 — Automation 执行引擎（2027-01-16）

> **目标**：让自动化规则从配置变成可执行能力。

| 任务 ID | 交付项 | 说明 |
|---------|--------|------|

| V32-AUTO-002 | 触发器 | issue created/updated/transitioned/commented/scheduled |
| V32-AUTO-003 | 条件引擎 | AND/OR、字段比较、JQL 条件 |
| V32-AUTO-004 | 动作引擎 | set field、assign、transition、comment、notify、webhook |
| V32-AUTO-005 | 执行日志 | 成功、失败、耗时、影响任务 |
| V32-AUTO-006 | 规则预览 | 显示将影响的 Issue 列表 |
| V32-AUTO-007 | 模板库 | 常见自动化模板一键创建 |
| V32-TEST | 测试 | 触发、条件、动作、失败重试 |

**交付物**：Automation 执行引擎、日志和模板库。

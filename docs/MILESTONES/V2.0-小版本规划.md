# 2.0 小版本发布计划 — Scheme 可配置版

> **更新时间**：2026-06-03
>
> **发布时间线**：2026-09-07 ~ 2026-11-28
>
> **路线原则**：V2.0 只做 Jira Scheme 主链路，不开发官方插件，不完善插件 SDK。扩展点继续作为契约预留。
>
> **主路线图**：见 [ROADMAP-CORE-FIRST.zh-CN.md](../ROADMAP-CORE-FIRST.zh-CN.md)。

---

## 2.0 核心目标

V2.0 的目标是把 V1.0 的固定任务系统升级为 Jira 式可配置平台，使项目可以配置自己的 Issue Type、Workflow、Screen、Field 和模板。

### 2.0 范围

| 模块 | 交付 |
|------|------|
| Issue Type | Epic / Story / Task / Bug / Subtask，自定义类型，类型方案 |
| Workflow | 状态、流转、条件、校验、后置动作，工作流方案 |
| Custom Field | 高频字段优先，逐步扩展字段类型 |
| Screen | 创建/编辑/查看屏幕，字段分组与排序 |
| Field Configuration | 隐藏、必填、只读、标签、默认值 |
| Project Template | 敏捷、Bug 跟踪、看板、空白项目模板 |
| JQL Lite | 基础查询语法、保存过滤器、常用函数 |
| 数据迁移 | V1.0 固定任务数据迁移到 V2.0 Scheme |

### 2.0 不做

| 功能 | 后续版本 |
|------|----------|
| 完整插件 SDK / CLI | V4.0 |
| 第三方插件热加载 | V4.0 |
| IM / CI/CD / WakaTime / 会议 | V4.0 官方插件池 |
| Automation 执行引擎 | V3.0 |
| Jira 导入向导 | V3.0 |

## 小版本规划

### 2.1 — Issue Type 与任务编号（2026-09-19）

> **目标**：先建立任务类型和任务编号体系，为后续工作流、屏幕和字段配置打基础。

| 任务 ID | 交付项 | 说明 |
|---------|--------|------|
| V20-TYPE-001 | Issue Type 数据模型 | issue_type、issue_type_scheme、scheme 关联表 |
| V20-TYPE-002 | 内置类型 | Epic、Story、Task、Bug、Subtask |
| V20-TYPE-003 | 自定义类型管理 | 新增、编辑、启用/禁用、排序、图标、颜色 |
| V20-TYPE-004 | 项目类型方案 | 项目可选择可用 Issue Type 集合 |
| V20-TYPE-005 | 任务编号 | PROJECT_KEY-1 格式，支持并发生成 |
| V20-TYPE-006 | 前端集成 | 创建/编辑/列表/看板显示类型和编号 |
| V20-TYPE-TEST | 测试 | 并发编号、类型禁用、项目方案边界 |

**交付物**：项目拥有独立 Issue Type Scheme，任务拥有稳定 issue key。

---

### 2.2 — Workflow 工作流引擎（2026-10-03）

> **目标**：把 V1.0 的固定状态流转升级为可配置工作流。

| 任务 ID | 交付项 | 说明 |
|---------|--------|------|
| V21-WF-001 | 工作流数据模型 | workflow、status、transition、condition、validator、post_function |
| V21-WF-002 | 内置工作流 | 简单任务流、Bug 流、敏捷 Story 流 |
| V21-WF-003 | 工作流方案 | Issue Type -> Workflow 映射 |
| V21-WF-004 | 流转引擎 | 可用流转、条件判断、校验、后置动作 |
| V21-WF-005 | 状态变更 API | transition API 返回可用流转和失败原因 |
| V21-WF-006 | 可视化编辑器 | 状态节点、流转箭头、条件/校验配置 |
| V21-WF-TEST | 测试 | 正常流转、非法流转、循环、全局流转 |

**交付物**：项目可配置不同 Issue Type 的工作流。

---

### 2.3 — Custom Field 高频字段（2026-10-17）

> **目标**：先实现 8 类高频字段，避免一次性追 25 类字段导致复杂度过高。

| 任务 ID | 交付项 | 说明 |
|---------|--------|------|
| V22-CF-001 | 字段定义模型 | custom_field_definition、context、option、value |
| V22-CF-002 | 高频字段类型 | 文本、长文本、数字、日期、单选、多选、用户、URL |
| V22-CF-003 | 字段上下文 | 全局、项目、Issue Type 上下文 |
| V22-CF-004 | 字段值存取 | 按字段类型路由到 EAV 存储列 |
| V22-CF-005 | 字段管理 UI | 字段列表、创建、编辑、选项管理 |
| V22-CF-006 | 动态渲染器 | 8 类字段组件和校验 |
| V22-CF-TEST | 测试 | 字段值读写、选项删除、搜索边界 |

**交付物**：用户可创建和使用高频自定义字段。

---

### 2.4 — Screen 与 Field Configuration（2026-10-31）

> **目标**：实现表单屏幕和字段行为控制，让不同操作看到不同字段。

| 任务 ID | 交付项 | 说明 |
|---------|--------|------|
| V23-SCR-001 | Screen 数据模型 | screen、screen_tab、screen_field、screen_scheme |
| V23-SCR-002 | Operation 映射 | create / edit / view / transition |
| V23-SCR-003 | 动态表单 API | 根据项目、类型、操作返回字段和行为 |
| V23-FC-001 | Field Config 模型 | hidden、required、readonly、label、default_value |
| V23-FC-002 | 字段行为校验 | 服务端统一校验必填、只读、隐藏字段 |
| V23-FE-001 | Screen 编辑器 | Tab、字段拖拽、排序 |
| V23-FE-002 | 动态任务表单 | 创建/编辑/详情切换为动态表单 |
| V23-TEST | 测试 | 不同类型、不同操作、不同项目组合测试 |

**交付物**：任务表单完全由 Screen + Field Config 驱动。

## 与扩展矩阵的关系

V2.0 只维护扩展点契约，不交付插件运行时完整能力。新增或稳定的扩展点包括：

| 扩展点 | 目的 |
|--------|------|
| Workflow Condition | 后续插件可提供自定义流转条件 |
| Workflow Validator | 后续插件可提供自定义校验 |
| Workflow Post Function | 后续插件可提供后置动作 |
| Field Renderer | 后续插件可提供自定义字段渲染器 |
| Field Searcher | 后续插件可提供自定义字段搜索器 |

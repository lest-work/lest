## Core vs Plugin 正确划分

> **Core 核心层**：平台必须的基础功能，所有插件依赖 Core 运行。
>
> **Plugin 插件层**：可选功能扩展，可按需安装/卸载/禁用。

### Core 核心层（V1.0 必须完成）

| 优先级 | 模块 | 说明 | 状态 |
|--------|------|------|------|
| P0 | 通知框架 | 事件总线 + WebSocket + 通知发送框架（不含具体渠道） | ❌ 脚手架 |
| P0 | 插件系统框架 | 插件生命周期管理 + SDK + 扩展点定义 | ❌ 脚手架 |
| P0 | 开放平台框架 | API Key 管理 + Webhook 接收框架 | ❌ 脚手架 |
| P0 | 系统管理精简 | 按 V2.0 PRD 精简为邮件/安全/审计（删除 OA 功能） | ⚠️ 部分 |

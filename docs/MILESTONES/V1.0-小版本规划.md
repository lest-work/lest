# 1.0 小版本发布计划 — Core-first

> **更新时间**：2026-06-04（最后一次状态核对）
>
> **发布时间线**：2026-06-04 ~ 2026-08-28
>
> **路线原则**：V1.0 只交付 Jira Core 基础能力和扩展矩阵契约，不交付会议、CI/CD、WakaTime、绩效、AI、IM 等官方插件。
>
> **主路线图**：见 [ROADMAP-CORE-FIRST.zh-CN.md](../ROADMAP-CORE-FIRST.zh-CN.md)。

## 1.0 核心目标

V1.0 的目标是让小团队可以真实使用 LEST 管项目、管任务、跑看板和 Sprint，并能完成私有化部署。此版本不追求插件生态，只为后续插件和商业能力预留稳定扩展点。

### 必须满足

| 领域 | 验收标准 | 状态 | 说明 |
| --- | --- | --- | --- |
| 稳定性 | 后端核心服务可编译、可启动；前端可构建 | ✅ | `mvn clean verify -DskipTests` 与 `npm run build` 均通过（2026-06-04） |
| 权限 | 所有项目/任务写操作都有项目级权限校验 | ✅ | Controller 加 `@RequireProjectRole`，任务写操作在 `TaskServiceImpl.requireTaskWithMembership` 兜底 @backend/apps/task/service/impl/TaskServiceImpl.java#102-912 |
| 核心流程 | 登录 → 建项目 → 加成员 → 建任务 → 看板拖拽 → 评论 → 通知 | ✅ | `docs/tests/core-e2e-smoke.http` 已覆盖管理员/成员双用户登录、项目创建与成员添加、任务创建与看板列移动、任务评论中 @mention 成员以及成员侧未读通知数量校验，可作为 V1.0 Core 冒烟脚本按顺序执行；建议发版前至少完整跑通一次并记录结果 |
| 数据一致性 | DDL、Domain、Mapper、API 返回结构一致 | ✅ | `TableDataInfo` 统一 `records/total` 字段；DTO 与 DDL 校验一致，如 `ProjectCreateRequest` @backend/common/core/web/page/TableDataInfo.java#19-77 @backend/apps/project/controller/request/ProjectCreateRequest.java#16 |
| 文档 | OpenAPI 与真实接口一致，部署文档可执行 | ⚠️ | 初始化 SQL 与模块部署文档已更新；OpenAPI 描述仍待核对完善 |

| V11-PRJ-002 | ✅ | 项目 Key | 创建项目后服务端自动生成唯一 Key（形如 `P{projectId}`，如 `P1`），作为 Issue Key 前缀并通过 API 返回 |

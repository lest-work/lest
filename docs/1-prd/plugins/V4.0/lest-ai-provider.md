# AI Provider 插件 PRD

> **版本归属**：V4.0.5
>
> **插件范围**：AI Provider、任务拆解、自然语言查询、风险识别的基础能力。

## 产品目标

把 AI 能力做成可替换 Provider，而不是写死在 Core。V4.0.5 先提供模型接入、权限边界、上下文注入和几类 Jira-like AI 场景，AI 助手与 AI 报表复用该 Provider 层。

## 功能范围

| 能力 | 说明 |
|------|------|
| Provider 管理 | OpenAI / Claude / Gemini / 本地模型 Provider 配置 |
| 模型路由 | 按场景选择模型、超时、重试、降级 |
| 成本控制 | 调用配额、单次最大 token、项目级开关 |
| 上下文注入 | Issue、Project、Sprint、Board 的最小必要上下文 |
| AI 任务拆解 | Epic / Story 拆解为可确认子任务树 |
| 自然语言查询 | 自然语言转 JQL / 过滤器草稿 |
| 风险识别 | 延期、阻塞、依赖、范围蔓延风险提示 |

## 不做

| 能力 | 归属 |
|------|------|
| 把 AI Provider 写入 V1~V3 Core | 禁止 |
| 自动执行高风险变更 | 必须二次确认，默认只生成建议 |
| 企业级 AI 审计和私有模型治理 | V5.0 商业增强 |

## 验收标准

1. 至少支持一个云端 Provider 和一个本地/自定义 Provider 配置。
2. 所有 AI 调用记录成本、耗时、模型、调用人和上下文来源。
3. AI 输出默认是建议草稿，涉及写操作必须用户确认。
4. Provider 不可用时，Core 主链路和非 AI 插件不受影响。

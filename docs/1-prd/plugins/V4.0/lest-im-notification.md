# IM 通知插件 PRD

> **版本归属**：V4.0.2
>
> **插件范围**：飞书、钉钉、企业微信通知插件 MVP。

## 产品目标

把 Core 的项目、Issue、Sprint、评论、提及、截止提醒等事件推送到企业 IM。V4.0 只做机器人/Webhook 通知，不做组织同步、审批流、SSO 或 IM 内工作台。

## 功能范围

| 功能 | 说明 |
|------|------|
| 飞书通知 | Webhook / 机器人消息推送、签名配置、测试发送 |
| 钉钉通知 | Webhook / 机器人消息推送、签名配置、测试发送 |
| 企业微信通知 | Webhook / 群机器人消息推送、测试发送 |
| 模板映射 | task/project/sprint/comment 事件映射为消息模板 |
| 配置页 | 项目级或全局配置、启停、发送测试、失败日志 |
| 失败处理 | 重试、降级到 Core 站内信、错误审计 |

## 不做

| 能力 | 归属 |
|------|------|
| 飞书/钉钉/企业微信组织同步 | V5.0 商业 Core 的企业身份/目录能力 |
| IM 审批流 | V5.0 企业流程能力或后续商业插件 |
| 把 IM 账号作为 Core 主账号体系 | V5.0 SSO / LDAP / SCIM |

## 事件范围

| Core 事件 | 默认接收人 |
|-----------|------------|
| Issue 创建 / 分配 / 状态变更 | 项目成员、Issue 关注人、经办人 |
| 评论 / @提及 | 被提及人、关注人 |
| Sprint 启动 / 完成 | Sprint 参与成员 |
| 截止提醒 | 经办人、关注人 |

## 验收标准

1. 三个 IM 平台均能完成配置、测试发送和启停。
2. Core 通知事件可按模板推送到已启用 IM 渠道。
3. IM 插件不可绕过 Core 权限读取无权访问的 Issue。
4. 推送失败可重试并写入插件日志，不影响 Core 通知发送。

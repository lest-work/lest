## 1. 概述

### 1.1 功能背景

团队绩效是 LEST Platform 帮助团队量化工作产出、提升效能的核心模块。通过整合任务工时、代码提交、MR 评审等多维度数据，平台为管理者提供客观的绩效参考，同时帮助团队成员了解自身工作状态。

### 1.2 功能目标

- 记录和统计任务工时（预估工时 vs 实际工时）
- 统计代码提交量和趋势
- 统计 MR/PR 评审参与度
- 计算团队和个人的关键效能指标（KPI）
- 支持自定义绩效指标和权重
- 提供可视化的绩效报表
- 支持绩效目标设定和追踪

### 1.3 目标用户

- **团队成员**：查看个人工作统计和目标达成
- **项目经理**：查看团队整体效能和成员贡献
- **管理层**：查看部门/公司级绩效报表
- **HR**：导出绩效数据用于考核

### 3.3 数据同步机制

#### 数据来源接口

| 数据源模块 | 获取方式 | 数据内容 | 同步频率 |
|-----------|---------|---------|---------|
| 任务模块 | API 调用 | 任务完成数、工时、准时率 | 实时 |
| 代码模块 | API 调用 | 提交数、代码量、MR 数 | 定时 |
| CI 模块 | API 调用 | 构建次数、成功率 | 定时 |
| 会议模块 | API 调用 | 参会次数、发言时长 | 定时 |
| 通知模块 | 事件订阅 | 关键事件通知 | 实时 |

#### 绩效指标计算说明

| 指标 | 计算公式 | 数据来源 |
|------|---------|---------|
| 完成任务数 | COUNT(task.status = 'completed' AND task.updated_at BETWEEN ...) | 任务模块 |
| 准时完成率 | COUNT(task.completed_at <= task.due_date) / COUNT(task.status = 'completed') | 任务模块 |
| 提交次数 | COUNT(DISTINCT commit_sha WHERE author_id = ...) | 代码模块 |
| 新增代码行数 | SUM(commit.additions) | 代码模块 |
| 评审响应时间 | AVG(review.created_at - mr.review_requested_at) | 代码模块 |
| 会议参与次数 | COUNT(DISTINCT meeting_id WHERE participant_id = ...) | 会议模块 |

### 3.3 绩效通知事件定义

| 事件编码 | 事件名称 | 触发时机 | 通知对象 | 优先级 |
|---------|---------|---------|---------|--------|
| PERFORMANCE_GOAL_ACHIEVED | 绩效目标达成 | 达成绩效目标时 | 用户本人 | P1 |
| PERFORMANCE_GOAL_PROGRESS | 绩效目标进度提醒 | 周期提醒（如每周） | 用户本人 | P2 |
| PERFORMANCE_TEAM_REPORT_READY | 团队绩效报告就绪 | 报告生成完成时 | 项目经理 | P1 |

## 3.X 绩效图表增强（V2.0/V3.0）

### 3.X.1 Burndown Chart 燃尽图（V2.0）

Sprint 燃尽图展示迭代内剩余工作量的变化趋势：

| 指标 | 说明 |
|------|------|
| 理想线 | 直线从总故事点递减到 0 |
| 实际线 | 每天结束时剩余故事点 |
| 偏差 | 实际与理想的差异 |

**触发时机：** Sprint 启动时开始计算，每日凌晨更新。

### 3.X.2 Burnup Chart 燃起图（V2.0）

与燃尽图互补，展示已完成工作量的增长：

| 指标 | 说明 |
|------|------|
| 理想线 | 直线从 0 递增到总故事点 |
| 实际完成线 | 每天结束时已完成故事点 |
| 范围线 | 展示总故事点的变化（范围变化时）|

### 3.X.3 Control Chart 控制图（V3.0）

展示每个 Issue 从创建到完成的周期时间分布：

| 指标 | 说明 |
|------|------|
| 平均周期时间 | 所有已完成 Issue 的平均时长 |
| 中位数周期时间 | 50% Issue 的完成时长 |
| P85/P95 周期时间 | 85%/95% Issue 的完成时长 |
| 周期时间趋势 | 按月/周统计周期时间变化 |

### 3.X.4 Cumulative Flow Diagram 累积流图（V3.0）

看板各列的任务数量随时间变化的可视化：

- X 轴：时间
- Y 轴：任务数量
- 每条彩带：代表一个状态（待办/进行中/已完成）
- 带宽越宽：该状态任务堆积越多
- 用途：识别流程瓶颈

### 3.X.5 Velocity Chart 速度图（V2.0）

展示团队每个 Sprint 完成的故事点：

| 指标 | 说明 |
|------|------|
| 实际速度 | Sprint 完成的故事点 |
| 平均速度 | 过去 N 个 Sprint 的平均值 |
| 预测速度 | 基于平均值预测未来 Sprint |
| 速度趋势 | 速度提升/下降趋势线 |

### 3.X.6 Team Capacity 团队容量（V2.0）

每个 Sprint 的团队可用容量计算：

| 输入项 | 说明 |
|--------|------|
| 团队成员数 | Sprint 开始时的成员数 |
| 可用工时/天 | 每人每天可用工时（扣除会议等）|
| Sprint 天数 | Sprint 的工作天数 |
| 假期/请假 | 成员请假/假期时间 |
| **可用容量** | **可用成员 × 可用工时 × 天数 - 假期** |

**容量不足预警：** 承诺故事点 > 可用容量时提示风险。

### 3.X.7 跨项目绩效报告（V3.0）

支持跨多个项目汇总绩效数据：

| 报告类型 | 内容 |
|---------|------|
| Portfolio 吞吐量 | 跨项目每周完成 Issue 数 |
| 团队贡献排行 | 按项目/按成员统计贡献 |
| 质量指标 | Bug 率（Bug 数 / 总 Issue 数）|
| 发布频率 | 每月/每周发布次数趋势 |

| 错误码 | 枚举常量 | HTTP 状态码 | 说明 |
|---------|----------|-------------|------|
| 14000 | `WORK_LOG_NOT_FOUND` | 404 | 工作日志不存在 |

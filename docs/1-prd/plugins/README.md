# 官方插件 -- Plugins

> **插件（Plugin）**是 LEST Platform 的功能扩展模块。所有非 Core 核心的功能都作为插件提供，可按需安装、启用和卸载。

## 设计原则

- **可插拔**：插件可随时安装/禁用/卸载，不影响 Core 运行
- **隔离性**：每个插件有独立的数据表前缀和资源空间
- **开源性**：所有官方插件开源（MIT License）
- **可发现性**：通过插件市场发现和安装插件

## Core vs Plugin 划分原则

|| 原则 | 说明 |
||------|------|
|| 核心依赖原则 | 功能是其他所有功能的前置依赖，必须在 Core |
|| 数据依赖原则 | 功能的数据模型是其他功能的数据基础，必须在 Core |
|| 扩展性原则 | 功能有无数种实现方式，放插件 |
|| 可插拔原则 | 功能可被完全移除而不影响 Core，放插件 |
|| 中国特色原则 | 中国特有的集成（钉钉/企微/飞书），必须是插件 |

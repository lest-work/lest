### 2.2 Project Labels（跨项目标签）

#### 2.2.1 功能说明

Labels（标签）是一种轻量级的跨项目分类方式。与 Component/Version 不同，Label 可以跨多个项目使用，用于标记 Issue 的特征（如 `frontend`、`bug`、`performance`、`needs-review`）。

#### 2.2.2 数据库设计

```sql
-- 标签定义
CREATE TABLE label (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    label_key       VARCHAR(64) NOT NULL UNIQUE COMMENT '标签唯一标识',
    name            VARCHAR(64) NOT NULL COMMENT '显示名称',
    color           VARCHAR(16) DEFAULT '#3B82F6' COMMENT '标签颜色',
    description     VARCHAR(256),
    use_count       INT DEFAULT 0 COMMENT '使用次数（缓存）',
    created_by      BIGINT NOT NULL,
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- Issue 标签关联
CREATE TABLE issue_label (
    issue_id        BIGINT NOT NULL,
    label_id        BIGINT NOT NULL,
    PRIMARY KEY (issue_id, label_id),
    FOREIGN KEY (issue_id) REFERENCES issue(id),
    FOREIGN KEY (label_id) REFERENCES label(id)
);
```

### 2.3 Project Clone（项目克隆）

#### 2.3.1 功能说明

支持将一个项目完整克隆为新项目，可选择克隆内容：

- 项目基本信息 + Scheme 配置（必选）
- Issue 数据（可选，含或不含历史评论/附件）
- 看板配置（可选）
- 组件/版本/迭代配置（可选）

## 3. API 设计（V3.0 新增接口）

| 接口 | 方法 | 说明 |
|------|------|------|
| `/project/{id}/labels` | GET/POST | 项目内标签列表/创建 |
| `/labels` | GET | 全局标签列表 |
| `/labels/{id}` | PUT/DELETE | 更新/删除标签 |
| `/labels/merge` | POST | 合并标签（A → B） |
| `/project/{id}/clone` | POST | 克隆项目 |
| `/project/{id}/archive` | PUT | 归档项目 |
| `/project/{id}/restore` | PUT | 恢复项目 |
| `/project/archived` | GET | 归档项目列表 |
| `/groups` | GET/POST | 用户组列表/创建 |
| `/groups/{id}` | PUT/DELETE | 用户组更新/删除 |
| `/groups/{id}/members` | GET/POST/DELETE | 组成员管理 |
| `/groups/{id}/usage` | GET | 用户组使用位置 |

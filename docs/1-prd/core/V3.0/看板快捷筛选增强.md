## 一、Quick Filters（看板快捷筛选）

### 1.1 功能说明

Quick Filters 是看板视图顶部的快捷筛选按钮，点击后立即过滤显示符合条件的卡片，无需输入 JQL。

### 1.2 默认 Quick Filters（开箱即用）

| 筛选名称 | JQL 条件 | 颜色 |
|---------|---------|------|
| **我的任务** | `assignee = currentUser() AND status NOT IN (Done, Closed)` | 蓝色 |
| **我的 Bug** | `assignee = currentUser() AND type = Bug AND status NOT IN (Done, Closed)` | 红色 |
| **高优先级** | `priority IN (P0, P1) AND status NOT IN (Done, Closed)` | 橙色 |
| **已延期** | `due < now() AND status NOT IN (Done, Closed)` | 红色 |
| **未分配** | `assignee IS EMPTY AND status NOT IN (Done, Closed)` | 灰色 |
| **今天到期** | `due = today()` | 红色 |
| **本周到期** | `due <= endOfWeek() AND due >= startOfWeek()` | 橙色 |

### 1.3 用户故事

#### US-QF-001: 使用 Quick Filter
**作为** 用户，**我希望** 点击 Quick Filter 快速筛选看板，**以便** 立即看到符合条件的任务。

验收标准：
- [ ] 看板顶部显示 Quick Filter 按钮横排
- [ ] 点击按钮，筛选条件应用到看板（仅显示匹配卡片）
- [ ] 激活的筛选按钮高亮显示
- [ ] 多个筛选可叠加（AND 关系）
- [ ] 点击「×」或再次点击按钮清除筛选

#### US-QF-002: 创建自定义 Quick Filter
**作为** 项目管理员，**我希望** 为看板创建自定义 Quick Filter，**以便** 团队成员快速访问常用筛选。

验收标准：
- [ ] 在看板设置中添加 Quick Filter
- [ ] 输入筛选名称 + JQL 条件
- [ ] 选择显示颜色（8 种可选颜色）
- [ ] 保存后，所有看板成员可见
- [ ] 支持拖拽排序筛选按钮顺序

#### US-QF-003: 我的 Quick Filters
**作为** 用户，**我希望** 将个人常用筛选保存为 Quick Filter，**以便** 快速访问。

验收标准：
- [ ] 任何 JQL 筛选可一键保存为 Quick Filter
- [ ] 个人筛选仅自己可见，不影响团队
- [ ] 个人筛选优先级高于项目筛选

### 1.4 数据库设计

```sql
-- 看板 Quick Filter 配置
CREATE TABLE board_quick_filter (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    board_id        BIGINT NOT NULL,
    filter_type     VARCHAR(16) NOT NULL COMMENT 'PROJECT/USER',
    filter_owner_id BIGINT COMMENT 'user_id（USER 类型时）',
    name            VARCHAR(128) NOT NULL,
    jql             VARCHAR(1024) NOT NULL,
    color           VARCHAR(16) DEFAULT '#3B82F6',
    sort_order      INT DEFAULT 0,
    is_enabled      BOOLEAN DEFAULT TRUE,
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (board_id) REFERENCES board(id)
);
```

## 二、看板卡片增强

### 2.1 卡片颜色规则

| 着色规则 | 说明 | 示例 |
|---------|------|------|
| 按优先级 | 卡片左边框颜色 = 优先级颜色 | P0 = 红色 |
| 按 Epic | 卡片顶边颜色 = Epic 颜色 | Epic-A = 紫色 |
| 按标签 | 卡片左边框颜色 = 标签配置颜色 | "frontend" = 蓝色 |
| 按负责人 | 卡片背景微调（不同人不同色调）| — |
| 按日期 | 截止日期临近时卡片变红 | 延期 = 红色 |

### 2.2 子任务折叠

## 三、列折叠与泳道增强

### 3.1 列折叠

| 操作 | 效果 |
|------|------|
| 点击列头「▼」| 列折叠为仅显示列名 + 卡片数量 |
| 点击「▶」| 列展开恢复 |
| 双击列头 | 折叠/展开切换 |
| 拖拽到折叠列 | 自动展开并放置 |

### 3.2 泳道增强

| 泳道类型 | 说明 |
|---------|------|
| 按 Epic | 每个 Epic 一个泳道，泳道颜色 = Epic 颜色 |
| 按负责人 | 每个成员一个泳道，显示成员头像 |
| 按标签 | 每个标签一个泳道 |
| 按组件 | 每个组件一个泳道 |
| 按版本 | 每个版本一个泳道 |

### 3.3 泳道路由

泳道路由确保 Epic 下所有 Story/Task 都在对应泳道内：

```
Epic A 泳道:
  └── Epic A
        ├── Story A1
        │     ├── Task A1-1
        │     └── Task A1-2
        └── Story A2
              └── Task A2-1

泳道路由规则：
  · 有 Epic Link 的任务 → 进入对应 Epic 泳道
  · 无 Epic Link 的任务 → 进入「无 Epic」泳道
  · Epic 泳道内的子任务 → 跟随父任务泳道
```

## 四、看板性能优化

| 优化项 | 说明 |
|--------|------|
| 虚拟滚动 | 超过 50 张卡片的列启用虚拟滚动，仅渲染可见卡片 |
| 懒加载 | 卡片详情（评论数/附件数）懒加载 |
| 卡片缓存 | 已渲染卡片在切换视图时缓存，不重复请求 |
| 拖拽优化 | 拖拽时使用 transform 而非 top/left，启用 GPU 加速 |
| WIP 限制检查 | 前端预检查 + 后端双重校验 |

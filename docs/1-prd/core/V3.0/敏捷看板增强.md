## 1. 概述

### 1.1 功能目标

V3.0 敏捷看板增强在 V2.0 基础看板之上，补全 Jira Software 的所有高级看板功能。

### 1.2 功能清单

| 功能 | 说明 | Jira 对应 |
|------|------|---------|
| 泳道（Swimlane） | 按负责人/史诗/标签等分组显示任务 | Swimlane |
| WIP 限制 | 限制每列最大任务数 | WIP Limit |
| 燃尽图（Burndown Chart） | 冲刺进度可视化 | Burndown Chart |
| 累积流图（CFD） | 工作流吞吐可视化 | Cumulative Flow Diagram |
| 速度图（Velocity Chart） | 团队速度历史趋势 | Velocity Chart |
| 控制图（Control Chart） | 任务周期时间分析 | Control Chart |
| 团队容量（Team Capacity） | 冲刺容量规划 | Team Capacity |
| 史诗进度（Epic Progress） | 史诗下所有任务的进度汇总 | Epic Progress |

## 2. 泳道（Swimlane）

### 2.1 用户故事

#### US-SWIMLANE-001: 按泳道分组看板
**作为** 团队成员，**我希望** 看板按泳道分组显示，**以便** 按负责人或史诗快速查看任务分布。

验收标准：
- [ ] 支持按负责人、史诗、标签、版本、自定义字段分组
- [ ] 每个泳道显示该组所有任务卡片
- [ ] 泳道可折叠/展开
- [ ] 泳道中的任务可跨列拖拽移动
- [ ] 空泳道可隐藏

---

## 3. WIP 限制（WIP Limit）

### 3.1 用户故事

#### US-WIP-001: 设置 WIP 限制
**作为** 看板管理员，**我希望** 限制每列的最大任务数，**以便** 控制团队工作负载。

验收标准：
- [ ] 在看板配置中设置每列的 WIP 限制
- [ ] 当列达到 WIP 限制时，列头显示警告色
- [ ] 超过 WIP 限制的任务拖拽时提示警告
- [ ] 允许临时突破 WIP 限制（带提示）

## 4. 燃尽图（Burndown Chart）

### 4.1 用户故事

#### US-BURNDOWN-001: 查看冲刺燃尽图
**作为** Scrum Master，**我希望** 查看冲刺燃尽图，**以便** 了解冲刺进度。

验收标准：
- [ ] X 轴：冲刺天数
- [ ] Y 轴：剩余故事点或任务数
- [ ] 显示理想线（直线下降）和实际线（实际剩余）
- [ ] 支持按故事点或按任务数两种模式
- [ ] 支持导出为图片

## 10. 数据库设计

```sql
-- board_swimlane（泳道配置）
CREATE TABLE board_swimlane (
  swimlane_id  BIGINT PK AUTO_INCREMENT,
  board_id     BIGINT NOT NULL,
  group_by     VARCHAR(32) NOT NULL,  -- assignee / epic / label / version / custom_field
  custom_field_id BIGINT,
  is_collapsed TINYINT DEFAULT 0,
  sort        INT DEFAULT 0,
  FOREIGN KEY (board_id) REFERENCES board(board_id)
);

-- board_column_wip（列 WIP 限制）
CREATE TABLE board_column_wip (
  wip_id       BIGINT PK AUTO_INCREMENT,
  board_column_id BIGINT NOT NULL,
  wip_limit    INT NOT NULL,           -- 0=不限制
  FOREIGN KEY (board_column_id) REFERENCES board_column(column_id)
);

-- sprint_burndown（冲刺燃尽图快照）
CREATE TABLE sprint_burndown (
  id           BIGINT PK AUTO_INCREMENT,
  sprint_id    BIGINT NOT NULL,
  record_date  DATE NOT NULL,
  total_points DECIMAL(10,2),          -- 总故事点
  remaining_points DECIMAL(10,2),       -- 剩余故事点
  total_count  INT,                     -- 总任务数
  remaining_count INT,                   -- 剩余任务数
  FOREIGN KEY (sprint_id) REFERENCES sprint(sprint_id),
  UNIQUE KEY (sprint_id, record_date)
);

-- team_capacity（团队容量）
CREATE TABLE team_capacity (
  capacity_id  BIGINT PK AUTO_INCREMENT,
  sprint_id    BIGINT NOT NULL,
  user_id      BIGINT NOT NULL,
  hours_per_day DECIMAL(4,1) DEFAULT 8.0,
  working_days INT,                     -- 可用工作日数
  total_hours  DECIMAL(6,1),
  FOREIGN KEY (sprint_id) REFERENCES sprint(sprint_id),
  UNIQUE KEY (sprint_id, user_id)
);
```

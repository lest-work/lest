## 一、仪表盘（Dashboard）

### 1.1 功能说明

个人仪表盘是用户登录后的首页，显示多个可配置的 Gadget（小组件），让用户一眼看到最关心的数据。

### 1.2 Dashboard 与 Gadget 的关系

```
Dashboard（仪表盘）
├── 我的仪表盘（个人创建，仅自己可见）
├── 项目仪表盘（项目级别，团队共享）
└── 全局仪表盘（管理员创建，平台级）

Gadget（小组件）
├── 内置 Gadget（系统提供）
└── 自定义 Gadget（通过插件扩展）
```

### 1.3 内置 Gadget 目录

| Gadget | 说明 | 数据来源 |
|--------|------|---------|
| **我的任务** | 当前用户负责的任务列表 | Issue API |
| **需要我处理的** | 待办/进行中的任务 | Issue API |
| **即将到期** | 截止日期在 7 天内的任务 | Issue API |
| **最近更新的** | 最近有变更的任务 | Activity API |
| **Sprint 进度** | 当前 Sprint 的燃尽图 | Sprint API |
| **项目统计** | 项目总任务/完成率 | Project API |
| **团队活动** | 团队最近的操作动态 | Activity API |
| **燃尽图** | Sprint 燃尽图 | Reports API |
| **速度图** | 最近几个 Sprint 的速度 | Reports API |
| **史诗进度** | Epic 完成百分比 | Epic API |
| **代码提交** | 最近 Git 提交 | Git API |
| **CI 构建状态** | 最近构建结果 | CI API |
| **快捷链接** | 自定义链接入口 | User Config |
| **统计数字** | 自定义数字指标 | Custom Query |
| **两列列表** | 左右双列任务列表 | Custom Query |
| **筛选结果** | 嵌入保存的过滤器结果 | Filter API |

#### US-DASH-002: 添加 Gadget

验收标准：

#### US-DASH-004: 保存仪表盘布局
**作为** 用户，**我希望** 保存仪表盘布局，**以便** 下次访问时保持一致。

### 1.5 数据库设计

```sql
-- Dashboard 配置
CREATE TABLE dashboard (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    dashboard_type  VARCHAR(16) NOT NULL COMMENT 'PERSONAL/PROJECT/GLOBAL',
    owner_id        BIGINT COMMENT '用户 ID（PERSONAL 时必填）',
    project_id      BIGINT COMMENT '项目 ID（PROJECT 时必填）',
    name            VARCHAR(128) NOT NULL,
    is_default      BOOLEAN DEFAULT FALSE,
    layout_config   JSON COMMENT 'Gadget 布局配置',
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- Gadget 配置
CREATE TABLE dashboard_gadget (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    dashboard_id    BIGINT NOT NULL,
    gadget_type     VARCHAR(64) NOT NULL COMMENT 'GADGET_TYPE_*',
    title           VARCHAR(128) COMMENT '自定义标题',
    position_x      INT NOT NULL COMMENT 'X 坐标',
    position_y      INT NOT NULL COMMENT 'Y 坐标',
    width           INT NOT NULL COMMENT '宽度（列数）',
    height          INT NOT NULL COMMENT '高度（行数）',
    config          JSON COMMENT 'Gadget 参数配置',
    refresh_interval INT DEFAULT 300 COMMENT '刷新间隔（秒）',
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (dashboard_id) REFERENCES dashboard(id)
);
```

## 二、日历视图（Calendar View）

### 2.1 功能说明

日历视图以月/周视图展示任务的截止日期和开始日期，方便用户直观地看到工作排期和截止点。

### 2.2 用户故事

#### US-CAL-001: 查看日历视图
**作为** 用户，**我希望** 在日历中查看任务的截止日期，**以便** 了解工作排期。

验收标准：
- [ ] 在项目/Backlog 中切换到「日历」视图
- [ ] 月视图：每个日期格显示当天截止的任务卡片
- [ ] 周视图：更详细地显示每个任务的截止时间
- [ ] 支持上下月/周切换
- [ ] 支持跳转到今天

#### US-CAL-002: 在日历中创建任务
**作为** 用户，**我希望** 在日历视图中快速创建任务，**以便** 方便地设置日期。

验收标准：
- [ ] 在日历格子中点击，直接创建截止日期为该天的任务
- [ ] 创建表单自动填充截止日期
- [ ] 支持拖拽任务卡片调整截止日期

#### US-CAL-003: 拖拽调整截止日期
**作为** 用户，**我希望** 在日历中拖拽任务卡片调整截止日期，**以便** 快速改期。

验收标准：
- [ ] 拖拽任务卡片到新日期格子
- [ ] 自动更新任务的截止日期字段
- [ ] 记录变更到操作历史

#### US-CAL-004: 日历筛选
**作为** 用户，**我希望** 在日历视图中按 Epic/负责人/类型筛选，**以便** 只看关心的任务。

验收标准：
- [ ] 支持按 Epic 筛选（只看特定 Epic 下的任务）
- [ ] 支持按负责人筛选
- [ ] 支持按 Issue Type 筛选
- [ ] 支持按标签筛选

## 三、过滤器订阅（Filter Subscription）

### 3.1 功能说明

过滤器订阅让用户将保存的 JQL 过滤器设置为定期自动通知，无需主动查看。

### 3.2 用户故事

#### US-FILTERSUB-001: 创建过滤器订阅
**作为** 用户，**我希望** 订阅一个过滤器，定期收到通知，**以便** 及时了解符合条件的任务动态。

验收标准：
- [ ] 保存过滤器时，勾选「订阅」
- [ ] 选择订阅频率：每天/每周/每月/自定义 Cron
- [ ] 选择通知时间（如每周一 9:00）
- [ ] 选择通知方式：站内信/邮件
- [ ] 订阅创建后按计划自动发送

#### US-FILTERSUB-002: 管理过滤器订阅
**作为** 用户，**我希望** 查看和管理我的过滤器订阅，**以便** 调整订阅设置。

验收标准：
- [ ] 在「我的过滤器」中显示所有订阅
- [ ] 显示：过滤器名称 / 频率 / 最近发送 / 状态
- [ ] 可暂停/恢复/编辑/删除订阅
- [ ] 可查看订阅发送历史

### 3.3 数据库设计

```sql
-- 过滤器订阅
CREATE TABLE filter_subscription (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id         BIGINT NOT NULL,
    filter_id       BIGINT NOT NULL COMMENT '关联的保存过滤器',
    schedule_type   VARCHAR(16) NOT NULL COMMENT 'DAILY/WEEKLY/MONTHLY/CUSTOM',
    schedule_cron   VARCHAR(64) COMMENT 'Cron 表达式（schedule_type=CUSTOM 时）',
    notify_time     TIME COMMENT '每天/每周的发送时间',
    notify_days     JSON COMMENT '每周几发送 [1,3,5]（周一三五）',
    notify_month_day INT COMMENT '每月第几天发送',
    notify_channel  VARCHAR(16) NOT NULL COMMENT 'IN_APP/EMAIL',
    is_enabled      BOOLEAN DEFAULT TRUE,
    last_sent_at    DATETIME,
    next_send_at    DATETIME,
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES sys_user(user_id),
    FOREIGN KEY (filter_id) REFERENCES saved_filter(id)
);

-- 订阅发送记录
CREATE TABLE subscription_send_log (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    subscription_id BIGINT NOT NULL,
    sent_at         DATETIME DEFAULT CURRENT_TIMESTAMP,
    recipient_count INT DEFAULT 1 COMMENT '接收人数',
    issue_count     INT COMMENT '符合条件的任务数',
    status          VARCHAR(16) COMMENT 'SUCCESS/FAILED',
    error_message   VARCHAR(512),
    FOREIGN KEY (subscription_id) REFERENCES filter_subscription(id)
);
```

## 四、国际化（i18n）架构设计

### 4.1 设计目标

LEST Platform 支持多语言，满足不同国家/地区用户的使用需求。

### 4.2 支持的语言

| 语言 | Locale Code | 状态 |
|------|-------------|------|
| 简体中文 | `zh-CN` | ✅ V1.0 默认 |
| 英文 | `en-US` | ✅ V2.0 |
| 繁体中文 | `zh-TW` | ⚠️ V2.0 |
| 日语 | `ja-JP` | ⚠️ V3.0 |
| 韩语 | `ko-KR` | ⚠️ V3.0 |

### 4.3 技术方案

```
i18n 架构：
┌─────────────────────────────────────────────────────────────┐
│  前端 (Vue i18n)                                        │
│  ├── locale/zh-CN.json  (简体中文翻译)                   │
│  ├── locale/en-US.json  (英文翻译)                     │
│  ├── locale/ja-JP.json  (日语翻译)                     │
│  └── locale/ko-KR.json  (韩语翻译)                     │
│  locale 文件结构：                                       │
│  {                                                      │
│    "issue.create": "创建任务",                          │
│    "issue.status.open": "待处理",                      │
│    "nav.projects": "项目",                               │
│    "btn.save": "保存",                                  │
│    "err.401": "无权限访问"                             │
│  }                                                      │
│  使用方式：                                              │
│  $t('issue.create')  // 在组件中                      │
│  i18n.t('issue.create')  // 在 JS 中                  │
└─────────────────────────────────────────────────────────────┘
```

### 4.4 数据库设计

```sql
-- 用户语言偏好
ALTER TABLE sys_user ADD COLUMN language VARCHAR(16) DEFAULT 'zh-CN';

-- 平台默认语言配置
CREATE TABLE platform_locale (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    locale_code     VARCHAR(16) NOT NULL UNIQUE,
    language_name   VARCHAR(64) NOT NULL COMMENT '中文名称',
    native_name     VARCHAR(64) NOT NULL COMMENT '本地名称',
    is_enabled      BOOLEAN DEFAULT TRUE,
    is_default      BOOLEAN DEFAULT FALSE,
    sort_order      INT DEFAULT 0,
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 翻译覆盖（允许平台为特定租户/用户覆盖翻译）
CREATE TABLE locale_override (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    locale_code     VARCHAR(16) NOT NULL,
    translation_key VARCHAR(256) NOT NULL,
    translation_value VARCHAR(1024) NOT NULL,
    scope           VARCHAR(16) DEFAULT 'PLATFORM' COMMENT 'PLATFORM/TENANT/USER',
    scope_id        BIGINT COMMENT 'tenant_id 或 user_id',
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE KEY uk_locale_key_scope (locale_code, translation_key, scope, scope_id)
);
```

## 五、暗黑模式（Dark Mode）

### 5.1 功能说明

支持深色和浅色两种主题模式，自动跟随系统偏好或手动切换。

### 5.2 用户故事

#### US-THEME-001: 切换主题
**作为** 用户，**我希望** 切换暗黑/亮色主题，**以便** 保护眼睛或适配个人偏好。

验收标准：
- [ ] 在「个人信息」中有「主题」选项
- [ ] 选项：跟随系统 / 亮色 / 暗黑
- [ ] 切换后立即生效
- [ ] 主题偏好保存到用户配置

### 5.3 CSS 变量设计

```css
/* Light Theme */
:root, [data-theme="light"] {
  --bg-primary: #FFFFFF;
  --bg-secondary: #F7F8FA;
  --bg-tertiary: #F0F2F5;
  --text-primary: #1A1A2E;
  --text-secondary: #6B7280;
  --border-color: #E5E7EB;
  --accent-color: #0052CC;
}

/* Dark Theme */
[data-theme="dark"] {
  --bg-primary: #0D0D0F;
  --bg-secondary: #1A1A1E;
  --bg-tertiary: #252529;
  --text-primary: #FFFFFF;
  --text-secondary: #9CA3AF;
  --border-color: #2D2D35;
  --accent-color: #4C9AFF;
}
```

## 1. 概述

### 1.1 V2.0 遗留的功能缺口

V2.0 完成了 Scheme 架构体系（EAV 自定义字段、Screen/Scheme、工作流引擎），但以下核心 Jira 功能仅有数据结构设计，缺少完整的用户故事和 API 设计：

| 功能 | V2.0 状态 | V3.0 补全目标 |
|------|----------|-------------|
| Issue Link | 有表结构，缺用户故事 | 完整 8 种链接类型 |
| Attachment | 缺失 | 完整附件体系 |
| Time Tracking | 缺失 | 预估/剩余/工时记录 |
| Sub-task | 基本父子关系 | 完整子任务 Issue Type |
| Automation 执行 | 有 trigger 数据结构 | 执行引擎 + 日志 |
| 回收站 | 缺失 | 软删除 + 恢复 |
| Vote / Share | 缺失 | 完整投票 + 分享体系 |
| 批量操作 | 缺失 | 批量修改/移动/删除 |
| 键盘快捷键 | 缺失 | 完整快捷键体系 |
| Automation Rule Templates | 缺失 | 规则模板库 |

此外，以下能力在 V2.0 PRD 中**已覆盖设计**，但若未在 V2.0 完整落地，则统一纳入 V3.0 作为 **Jira 100% 功能收口项**实现并验收：

- 批量操作（Batch Operations）
- 投票/分享（Vote / Share）
- 键盘快捷键（Shortcuts）
- 自动化规则模板库（Automation Rule Templates）

### 1.2 V3.0 补全策略

V3.0 **不改变 V2.0 的 Scheme 架构**，而是在其基础上补充缺失的用户故事和 API 设计，确保每个 Jira 核心功能都有对应的实现。

## 2. Issue Link（任务关联）

### 2.1 功能说明

Jira Issue Link 允许用户在一个 Issue 上关联另一个 Issue，形成需求网络。每个链接有方向性和链接类型。

### 2.2 链接类型（8 种）

| 链接类型 key | 名称 | 方向 | 说明 |
|-------------|------|------|------|
| `blocks` | 阻塞 | 单向 | 当前任务阻塞目标任务 |
| `is_blocked_by` | 被阻塞 | 单向 | 当前任务被目标任务阻塞 |
| `duplicates` | 重复 | 单向 | 当前任务与目标任务重复 |
| `is_duplicated_by` | 被重复 | 单向 | 当前任务被目标任务重复 |
| `relates_to` | 相关 | 双向 | 两个任务相关 |
| `causes` | 导致 | 单向 | 当前任务导致目标任务 |
| `is_caused_by` | 被导致 | 单向 | 当前任务被目标任务导致 |
| `clones` | 克隆 | 单向 | 当前任务克隆自目标任务 |

### 2.3 用户故事

#### US-ISSUELINK-001: 创建任务链接
**作为** 任务负责人，**我希望** 将当前任务与其他任务关联，**以便** 表达任务之间的关系。

验收标准：
- [ ] 在任务详情页点击"添加关联"按钮
- [ ] 选择链接类型（blocks / relates_to / duplicates 等）
- [ ] 搜索并选择目标任务（支持 JQL 搜索）
- [ ] 自动发送通知给被关联任务的负责人
- [ ] 链接记录到任务操作历史

#### US-ISSUELINK-002: 查看任务链接
**作为** 团队成员，**我希望** 查看一个任务的所有链接，**以便** 了解任务之间的关系网络。

验收标准：
- [ ] 在任务详情页"关联"Tab 中显示所有链接
- [ ] 按链接类型分组显示（正向/反向）
- [ ] 点击链接跳转到目标任务
- [ ] 显示链接的创建人和创建时间
- [ ] 支持筛选特定类型的链接

#### US-ISSUELINK-003: 批量管理链接
**作为** 项目经理，**我希望** 批量创建或删除任务链接，**以便** 快速建立任务网络。

验收标准：
- [ ] 支持 CSV 导入批量创建链接
- [ ] 支持批量删除选中的链接
- [ ] 批量操作记录到审计日志

### 2.4 数据库设计

```sql
-- issue_link_type（链接类型）
CREATE TABLE issue_link_type (
  link_type_id   BIGINT PK AUTO_INCREMENT,
  link_key       VARCHAR(32) NOT NULL UNIQUE,  -- blocks / relates_to
  name           VARCHAR(64) NOT NULL,           -- 阻塞 / 相关
  inward         VARCHAR(64),                     -- 反向描述：被阻塞
  outward        VARCHAR(64),                     -- 正向描述：阻塞
  is_system      TINYINT DEFAULT 0,              -- 1=内置不可删除
  is_active      TINYINT DEFAULT 1,
  created_at     DATETIME
);

-- issue_link（任务链接）
CREATE TABLE issue_link (
  link_id        BIGINT PK AUTO_INCREMENT,
  source_issue_id BIGINT NOT NULL,               -- 源任务
  target_issue_id BIGINT NOT NULL,               -- 目标任务
  link_type_id   BIGINT NOT NULL,                -- 链接类型
  created_by     BIGINT NOT NULL,
  created_at     DATETIME,
  FOREIGN KEY (source_issue_id) REFERENCES issue(id),
  FOREIGN KEY (target_issue_id) REFERENCES issue(id),
  FOREIGN KEY (link_type_id) REFERENCES issue_link_type(link_type_id)
);
```

Jira Time Tracking 包含三个核心概念：预估工时（Estimate）、剩余工时（Remaining Estimate）、已记录工时（Time Spent）。

验收标准：
- [ ] 在任务详情页设置预估工时（格式：1w 2d 3h 4m）
- [ ] 支持快速选择预设值（1h / 2h / 4h / 8h / 1d / 3d）
- [ ] 预估工时显示在任务卡片和列表中

#### US-TIME-002: 记录已工时
**作为** 任务负责人，**我希望** 记录在任务上花费的时间，**以便** 追踪实际工作量。

验收标准：
- [ ] 在任务详情页记录工时（工时 + 日期）
- [ ] 支持添加工时说明
- [ ] 记录后自动更新剩余工时 = 预估工时 - 已记录工时
- [ ] 剩余工时为负时给出警告
- [ ] 工时记录支持编辑和删除

#### US-TIME-003: 进度可视化
**作为** 项目经理，**我希望** 查看任务的工时进度，**以便** 了解任务完成情况。

验收标准：
- [ ] 任务卡片显示进度条（已记录/预估）

### 4.3 数据库设计

```sql
-- 在 issue 表中增加时间追踪字段（V3.0）
ALTER TABLE issue ADD COLUMN estimate_seconds BIGINT;         -- 预估工时（秒）
ALTER TABLE issue ADD COLUMN remaining_seconds BIGINT;        -- 剩余工时（秒）
ALTER TABLE issue ADD COLUMN time_spent_seconds BIGINT DEFAULT 0;  -- 已记录工时（秒）

-- worklog（工时记录）
CREATE TABLE worklog (
  worklog_id     BIGINT PK AUTO_INCREMENT,
  issue_id       BIGINT NOT NULL,
  user_id        BIGINT NOT NULL,
  started_at     DATETIME NOT NULL,                 -- 工作开始时间
  duration_seconds BIGINT NOT NULL,                  -- 工时时长（秒）
  description    VARCHAR(512),                       -- 工时说明
  created_at     DATETIME,
  updated_at     DATETIME,
  is_deleted     TINYINT DEFAULT 0,
  FOREIGN KEY (issue_id) REFERENCES issue(id)
);
```

## 6. Automation 执行引擎（补全 V2.0）

### 6.1 功能说明

V2.0 已设计 Automation Rule 的数据结构（trigger/condition/action），V3.0 需要实现执行引擎。

### 6.2 执行引擎设计

```
触发事件发生
    │
    ▼
规则引擎扫描所有启用的规则
    │
    ▼
匹配 trigger_type（issue_created / field_changed / scheduled）
    │
    ▼
评估 conditions（AND/OR 条件树）
    │
    ├── 条件不满足 → 跳过
    │
    ▼
执行 actions（按 execute_order）
    │
    ├── SET_FIELD / SEND_NOTIFICATION / CREATE_SUBTASK 等
    │
    ▼
记录执行日志（automation_execution_log）
    │
    ▼
出错则按 fail_action 处理（skip / error）
```

### 6.3 用户故事

#### US-AUTO-001: 创建自动化规则
**作为** 项目管理员，**我希望** 创建自动化规则，**以便** 减少重复性操作。

验收标准：
- [ ] 从预设模板创建规则（"任务创建时自动分配"等）
- [ ] 手动配置 trigger / conditions / actions
- [ ] 规则预览：显示满足条件时会影响的 Issue 列表
- [ ] 规则可启用/禁用

#### US-AUTO-002: 查看规则执行日志
**作为** 管理员，**我希望** 查看自动化规则的执行日志，**以便** 排查问题和审计。

验收标准：
- [ ] 显示每个规则的执行历史
- [ ] 日志包含：触发时间、影响的问题、执行的动作、是否成功
- [ ] 失败的执行显示错误原因
- [ ] 支持按规则/按问题/按时间筛选

### 6.4 数据库设计

```sql
-- automation_rule（自动化规则）
CREATE TABLE automation_rule (
  rule_id        BIGINT PK AUTO_INCREMENT,
  project_id     BIGINT,                           -- NULL=全局规则
  name           VARCHAR(128) NOT NULL,
  description    TEXT,
  trigger_type   VARCHAR(32) NOT NULL,              -- issue_created / field_changed / scheduled
  trigger_config JSON,
  conditions     JSON,                              -- 条件树
  actions       JSON,                              -- 动作列表
  is_enabled    TINYINT DEFAULT 1,
  created_by     BIGINT,
  created_at     DATETIME,
  updated_at     DATETIME
);

-- automation_execution_log（执行日志）
CREATE TABLE automation_execution_log (
  log_id         BIGINT PK AUTO_INCREMENT,
  rule_id        BIGINT NOT NULL,
  trigger_event  VARCHAR(64) NOT NULL,              -- 触发事件描述
  issue_id       BIGINT,                            -- 关联的问题
  executed_by    VARCHAR(32),                       -- system / user_id
  actions_executed JSON,                            -- 执行的动作
  status         VARCHAR(16),                       -- success / failed / skipped
  error_message  TEXT,
  executed_at    DATETIME,
  FOREIGN KEY (rule_id) REFERENCES automation_rule(rule_id)
);
```

- [ ] 投票后用户自动 Watch 该任务

#### US-VOTE-002: 查看投票列表
**作为** 项目经理，**我希望** 查看一个项目中投票最多的任务，**以便** 了解团队最关注的问题。

### 8.4 数据库设计

```sql
-- vote（投票）
CREATE TABLE vote (
    issue_id    BIGINT NOT NULL,
    user_id     BIGINT NOT NULL,
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (issue_id, user_id),
    FOREIGN KEY (issue_id) REFERENCES issue(id),
    FOREIGN KEY (user_id) REFERENCES sys_user(user_id)
);

-- share（分享）
CREATE TABLE share (
    id          BIGINT PRIMARY KEY AUTO_INCREMENT,
    issue_id    BIGINT NOT NULL,
    share_type  VARCHAR(16) NOT NULL COMMENT 'USER/GROUP',
    grantee_id  BIGINT NOT NULL COMMENT 'user_id / group_id',
    perm_level  VARCHAR(16) NOT NULL COMMENT 'VIEW/EDIT/COMMENT',
    shared_by   BIGINT NOT NULL,
    shared_at   DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (issue_id) REFERENCES issue(id)
);
```

## 9. Batch Operations（批量操作）

### 9.1 功能说明

批量操作允许用户同时对多个任务执行同一操作，显著提升工作效率。

## 11. Automation Rule Templates（自动化规则模板库）

### 11.1 功能说明

规则模板库为用户提供预设的自动化规则，开箱即用，无需从零配置。

### 11.2 内置模板列表

| 模板名称 | 触发器 | 条件 | 动作 | 适用场景 |
|---------|--------|------|------|---------|
| **自动分配 Bug** | Bug 创建时 | — | 分配给项目负责人 | 新 Bug 自动指派 |
| **高优先级任务通知** | 优先级变更 | 新优先级 = P1/P2 | 发送通知给项目管理员 | 高优任务及时知晓 |
| **超时提醒** | 定时（每天） | 任务状态 = In Progress 超过 3 天 | 发送提醒给负责人 | 防止任务卡住 |
| **子任务全部完成后关闭父任务** | 子任务状态变更 | 所有子任务 = Done | 将父任务设为 Done | 自动收尾 |
| **Story 完成时通知测试** | Story 状态变更 | 新状态 = Done | 通知 QA 角色 | 测试及时介入 |
| **Epic 进度更新** | 定时（每天） | — | 更新 Epic 进度百分比 | 自动追踪 Epic |
| **新评论时通知被@人** | 评论添加 | 评论中包含 @用户名 | 发送通知 | @提及提醒 |
| **任务延期预警** | 定时（每天） | 截止日期 < 今天 且状态 ≠ Done | 发送告警 | 延期预警 |
| **Sprint 结束时归档** | Sprint 结束 | — | 归档未完成的 Issue | 保持 Sprint 清洁 |
| **值岗工程师自动分配** | 值班表触发 | — | 按值班表分配任务 | 轮流值班 |

## 13. 任务导入与导出（Import / Export）

### 13.1 功能说明

任务导入导出支持从 Excel/CSV 批量迁移数据，以及将任务列表导出为 Excel 离线查看或上报。

### 13.2 数据库设计

```sql
-- 导入任务记录
CREATE TABLE issue_import (
  id              BIGINT PK AUTO_INCREMENT,
  import_id       VARCHAR(36) NOT NULL,
  project_id      BIGINT NOT NULL,
  creator_id      BIGINT NOT NULL,
  file_name       VARCHAR(255) NOT NULL,
  file_url        VARCHAR(500),
  total_rows      INT,
  success_count   INT DEFAULT 0,
  fail_count      INT DEFAULT 0,
  status          VARCHAR(20),
  created_at      DATETIME,
  finished_at     DATETIME,
  FOREIGN KEY (project_id) REFERENCES project(id),
  FOREIGN KEY (creator_id) REFERENCES sys_user(id)
);

-- 导入错误行记录
CREATE TABLE issue_import_error (
  id              BIGINT PK AUTO_INCREMENT,
  import_id       VARCHAR(36) NOT NULL,
  row_number      INT,
  field_name      VARCHAR(100),
  field_value     VARCHAR(500),
  error_message   VARCHAR(500),
  created_at      DATETIME
);

-- 导出任务记录
CREATE TABLE issue_export (
  id              BIGINT PK AUTO_INCREMENT,
  export_id       VARCHAR(36) NOT NULL,
  project_id      BIGINT,
  creator_id      BIGINT NOT NULL,
  filters         TEXT,
  selected_fields TEXT,
  file_name       VARCHAR(255),
  file_url        VARCHAR(500),
  status          VARCHAR(20),
  created_at      DATETIME,
  expired_at      DATETIME,
  FOREIGN KEY (creator_id) REFERENCES sys_user(id)
);
```

### 13.3 用户故事

#### US-IMPORT-001: Excel 导入任务
**作为** 项目管理员，**我希望** 从 Excel/CSV 批量导入任务，**以便** 从其他系统迁移数据。

验收标准：
- [ ] 支持 Excel (.xlsx) 和 CSV 格式上传
- [ ] 支持字段映射：标题/描述/负责人/状态/优先级/截止日期/故事点
- [ ] 导入前预览并支持修正错误行
- [ ] 导入后显示成功/失败统计
- [ ] 重复任务检测（按标题+项目判断，可选跳过或覆盖）

#### US-IMPORT-002: 导出任务列表
**作为** 团队成员，**我希望** 导出任务列表为 Excel，**以便** 离线查看或上报。

验收标准：
- [ ] 支持导出当前筛选结果
- [ ] 支持字段选择（勾选要导出的列）
- [ ] 导出文件自动命名为"项目名_导出时间.xlsx"

### 13.4 API 设计

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| POST | `/projects/{projectId}/issues/import` | 上传导入文件 | project_admin |
| GET | `/projects/{projectId}/issues/import/{importId}` | 获取导入状态和错误 | project_admin |
| GET | `/projects/{projectId}/issues/import/{importId}/errors` | 获取错误行列表 | project_admin |
| POST | `/projects/{projectId}/issues/import/{importId}/retry/{rowNumber}` | 重试失败行 | project_admin |
| POST | `/projects/{projectId}/issues/export` | 创建导出任务 | project_member |
| GET | `/projects/{projectId}/issues/export/{exportId}` | 获取导出状态 | creator |
| GET | `/projects/{projectId}/issues/export/{exportId}/download` | 下载导出文件 | creator |

# 审计日志 — V5.0 PRD

> **模块**: Core V5.0 | **许可**: 🔒 商业版

## 1. 产品概述

### 1.1 目标

满足等保/ISO27001 合规要求，记录所有操作行为的完整审计日志，提供日志查询和合规报表导出。

## 2. 核心功能

|| 功能 | 说明 | 优先级 |
||------|------|--------|
| 操作审计 | 记录所有用户操作（登录/数据变更/配置修改） | P0 |
| 审计日志查询 | 分页查询 + 多条件筛选 + 导出 | P0 |
| 合规报表 | 按时间/用户/操作类型聚合 | P1 |
| 实时告警 | 敏感操作实时通知管理员 | P1 |
| 日志归档 | 历史日志归档策略 | P2 |

## 3. 审计事件范围

|| 模块 | 事件 |
||------|------|
| 认证 | LOGIN / LOGOUT / LOGIN_FAIL / SSO_LOGIN |
| 用户管理 | USER_CREATE / USER_UPDATE / USER_DELETE / USER_DISABLE |
| 项目管理 | PROJECT_CREATE / PROJECT_UPDATE / PROJECT_DELETE / PROJECT_ARCHIVE |
| 任务管理 | TASK_CREATE / TASK_UPDATE / TASK_DELETE / TASK_STATUS_CHANGE / TASK_ASSIGN |
| 权限管理 | ROLE_ASSIGN / ROLE_REVOKE / PERMISSION_CHANGE |
| 配置变更 | CONFIG_CREATE / CONFIG_UPDATE / CONFIG_DELETE |
| 数据导出 | DATA_EXPORT |

## 4. 数据库设计

```sql
CREATE TABLE audit_log (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id         BIGINT,
    username        VARCHAR(128),
    tenant_id       BIGINT,
    action          VARCHAR(64) NOT NULL COMMENT 'ACTION_*',
    module          VARCHAR(32) NOT NULL,
    resource_type   VARCHAR(32),
    resource_id     VARCHAR(64),
    detail          JSON COMMENT '变更前后数据',
    ip_address      VARCHAR(45),
    user_agent      VARCHAR(512),
    result          VARCHAR(16) COMMENT 'SUCCESS/FAIL',
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_tenant_time (tenant_id, created_at),
    INDEX idx_user_time (user_id, created_at),
    INDEX idx_action (action, created_at)
);
```

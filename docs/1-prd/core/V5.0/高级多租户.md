# 高级多租户 — V5.0 PRD

> **模块**: Core V5.0 | **许可**: 🔒 商业版

## 1. 产品概述

### 1.1 目标

在基础多租户之上，实现 Schema 级别的完全数据隔离，提供租户管理控制台和配额控制。

## 2. 核心功能

|| 功能 | 说明 | 优先级 |
||------|------|--------|
| Schema 隔离 | 每租户独立数据库 Schema | P0 |
| 认证隔离 | 租户专属 SSO/LDAP | P0 |
| 存储隔离 | MinIO bucket 按租户隔离 | P0 |
| 消息隔离 | Kafka topic 按租户隔离 | P0 |
| 租户管理控制台 | 平台管理员统一管理 | P0 |
| 租户配额控制 | 资源使用上限 | P0 |

## 3. 隔离架构

```
┌──────────────────────────────────────────────────────┐
│                    LEST Platform                         │
│  ┌────────────┐  ┌────────────┐  ┌────────────┐   │
│  │ 租户 A     │  │ 租户 B     │  │ 租户 C     │   │
│  │ Schema A  │  │ Schema B  │  │ Schema C  │   │
│  │ Bucket A  │  │ Bucket B  │  │ Bucket C  │   │
│  │ Topic A  │  │ Topic B  │  │ Topic C  │   │
│  └────────────┘  └────────────┘  └────────────┘   │
└──────────────────────────────────────────────────────┘
```

## 4. 数据库设计

```sql
-- Schema 初始化模板
CREATE TABLE tenant_schema (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    tenant_id       BIGINT NOT NULL UNIQUE,
    schema_name     VARCHAR(64) NOT NULL UNIQUE,
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE tenant_quota (
    id                  BIGINT PRIMARY KEY AUTO_INCREMENT,
    tenant_id           BIGINT NOT NULL,
    resource_type       VARCHAR(32) NOT NULL COMMENT 'USER/STORAGE/API_CALL',
    limit_value        BIGINT NOT NULL,
    used_value          BIGINT DEFAULT 0,
    period_type        VARCHAR(16) COMMENT 'MONTHLY/TOTAL',
    updated_at         DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE(tenant_id, resource_type)
);
```

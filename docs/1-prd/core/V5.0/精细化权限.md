# 精细化权限 — V5.0 PRD

> **模块**: Core V5.0 | **许可**: 🔒 商业版

## 1. 产品概述

### 1.1 目标

在项目级角色（Admin/Member/Viewer）基础上，提供行级数据权限控制和字段级数据屏蔽能力。

## 2. 核心功能

|| 功能 | 说明 | 优先级 |
||------|------|--------|
| 行级权限 | 基于字段值控制数据访问 | P0 |
| 字段级屏蔽 | 敏感字段对特定角色隐藏 | P0 |
| 权限继承 | 子项目继承父项目权限 | P1 |
| 动态权限规则 | 基于用户属性动态计算权限 | P1 |
| 权限审计 | 权限变更记录和查询 | P2 |

## 3. 权限规则引擎

```
请求: GET /api/issues?project_id=123
  ↓
权限规则引擎
  ↓
1. 获取用户角色: Member
2. 获取项目权限规则:
   - 所有人可看: status != "秘密"
   - 只有成员可看: 所有字段
   - 只有成员可编辑: assignee = 当前用户
3. 注入 WHERE 条件:
   WHERE project_id = 123 AND status != '秘密'
  ↓
返回过滤后的数据
```

## 4. 数据库设计

```sql
CREATE TABLE permission_rule (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    resource_type   VARCHAR(32) NOT NULL COMMENT 'PROJECT/TASK/ISSUE',
    resource_id     VARCHAR(64),
    role_id        VARCHAR(32) NOT NULL,
    rule_type      VARCHAR(16) NOT NULL COMMENT 'ROW_LEVEL/FIELD_LEVEL',
    field_name     VARCHAR(64),
    operator       VARCHAR(16) NOT NULL COMMENT 'EQ/NE/LIKE/IN/GT/LT',
    value          VARCHAR(256) COMMENT '静态值或 ${user.xxx} 动态值',
    effect         VARCHAR(8) NOT NULL COMMENT 'ALLOW/DENY',
    priority       INT DEFAULT 0,
    created_at     DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE field_mask (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    resource_type   VARCHAR(32) NOT NULL,
    field_name     VARCHAR(64) NOT NULL,
    role_id        VARCHAR(32) NOT NULL,
    mask_type      VARCHAR(16) NOT NULL COMMENT 'HIDE/REDACT/SHOW_LAST4',
    created_at     DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

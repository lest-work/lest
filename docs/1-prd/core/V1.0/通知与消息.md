# 通知与消息 PRD — Core V1.0

> **Jira 映射**：Jira Notifications 的基础站内信和邮件通知。
>
> **开发口径**：V1.0 Core 只内置站内信、WebSocket 和 SMTP 邮件。飞书、钉钉、企业微信、Slack、会议通知、CI/CD 通知、WakaTime、AI、绩效等全部由 V4.0 插件或 V5.0 企业能力提供。

## 1. V1.0 范围

| 模块 | V1.0 必须交付 | 小版本 |
|------|---------------|--------|
| 通知事件 | Core 项目、Issue、评论、Sprint、附件事件 | 1.5 |
| 站内信 | 列表、未读数、标记已读、全部已读、跳转关联对象 | 1.5 |
| WebSocket | 用户级实时推送、重连恢复 | 1.5 |
| 邮件 | SMTP 配置、测试发送、核心事件邮件模板 | 1.5 |
| 通知偏好 | 用户关闭邮件、站内信免打扰 | 1.5 P1 |
| 扩展契约 | NOTIFICATION_CHANNEL 接口定义，不实现插件运行时 | 1.6 |

## 2. Core 通知事件

| 事件 | 触发时机 | 默认接收人 | 渠道 |
|------|----------|------------|------|
| project.created | 项目创建 | 创建者 | 站内信 |
| project.member_added | 成员加入项目 | 新成员、项目 Admin | 站内信 / 邮件 |
| project.member_role_changed | 成员角色变更 | 被调整成员 | 站内信 |
| sprint.started | Sprint 启动 | 项目成员 | 站内信 |
| sprint.completed | Sprint 完成 | 项目成员 | 站内信 |
| task.created | Issue 创建 | 负责人、关注者 | 站内信 |
| task.assigned | Issue 指派 | 新负责人 | 站内信 / 邮件 |
| task.status_changed | 状态变更 | 负责人、关注者 | 站内信 |
| task.commented | 评论 | 被 @ 提及者、负责人、关注者 | 站内信 / 邮件 |
| task.due_soon | 即将到期 | 负责人 | 站内信 / 邮件 |
| task.overdue | 已逾期 | 负责人、创建者 | 站内信 / 邮件 |
| task.dependency_unblocked | 依赖解除 | 被解除阻塞的 Issue 负责人 | 站内信 |
| attachment.added | 附件上传 | 负责人、关注者 | 站内信 |

## 3. 用户故事

### US-NOT-001 查看站内信

验收标准：
- [ ] 显示通知标题、摘要、类型、时间、关联对象
- [ ] 支持分页
- [ ] 支持未读数红点
- [ ] 点击通知可跳转到项目、Issue 或 Sprint

### US-NOT-002 标记已读

验收标准：
- [ ] 可单条标记已读
- [ ] 可全部标记已读
- [ ] 已读状态实时更新未读数

### US-NOT-003 实时推送

验收标准：
- [ ] WebSocket 按用户维度推送
- [ ] 离线期间产生的通知可在重连后从站内信拉取
- [ ] 推送失败不影响业务操作

### US-NOT-004 邮件通知

验收标准：
- [ ] 平台管理员可配置 SMTP
- [ ] 可发送测试邮件
- [ ] 指派、@ 提及、即将到期、逾期默认发送邮件
- [ ] 用户可关闭非关键邮件

### US-NOT-005 通知偏好

验收标准：
- [ ] 用户可设置邮件开关
- [ ] 用户可设置站内信免打扰时段
- [ ] 关键安全事件不受免打扰影响

## 1. 概述

### 1.1 功能背景

通知与消息是 LEST Platform 实现团队协作沟通的核心模块。平台需要支持多种通知渠道（站内信、邮件、钉钉、企业微信、飞书），确保团队成员及时获取重要事件通知，同时提供消息聚合和已读未读管理。

### 1.2 功能目标

- 提供站内消息通知
- 支持多渠道通知（邮件、钉钉、企业微信、飞书）
- 支持通知模板配置
- 支持通知规则配置（按事件类型、用户角色、项目订阅）
- 提供消息聚合和去重
- 支持消息免打扰设置
- 记录通知送达状态

### 1.3 目标用户

- **所有用户**：接收各类业务事件通知
- **项目管理员**：配置项目通知规则
- **平台管理员**：配置全局通知渠道和模板

## 2. 用户故事

### 2.1 站内消息

#### US-001: 查看消息列表
**作为** 用户，**我希望** 查看我的站内消息，**以便** 了解需要关注的事项。

验收标准：
- [ ] 显示消息列表（标题、内容、类型、发送时间）
- [ ] 支持按类型筛选（任务/MR/系统/项目）
- [ ] 支持标记已读/未读
- [ ] 支持一键全部已读
- [ ] 支持搜索消息
- [ ] 支持分页加载
- [ ] 未读消息显示数量红点

验收标准：

### 3.4 通知事件订阅接口

```yaml
# 各业务模块向通知模块订阅事件的接口

POST /notification/subscribe
Authorization: Bearer {accessToken}
Content-Type: application/json

请求:
{
  "module": "task",
  "events": [
    "TASK_ASSIGNED",
    "TASK_STATUS_CHANGED",
    "TASK_COMMENTED"
  ],
  "channels": ["inApp", "email"],
  "filter": {
    "projectIds": [1, 2, 3]
  }
}

响应:
{
  "code": 200,
  "message": "success"
}
```

### 3.X 条件通知（V2.0）

Jira 支持基于条件的通知规则，V2.0 实现：

| 条件类型 | 示例 | 说明 |
|---------|------|------|
| 字段值条件 | Priority = P0 | 只有 P0 级别才发送通知 |
| Issue Type 条件 | Type = Bug | 只对 Bug 类型发送通知 |
| 状态变更条件 | from=进行中, to=已完成 | 只在特定状态流转时通知 |
| 角色条件 | Assignee = 项目负责人 | 通知特定角色 |
| 字段变更条件 | Priority changed | 只在字段变更时通知 |

**条件通知配置示例：**

```json
{
  "name": "P0 Bug 紧急通知",
  "trigger": "issue.created",
  "conditions": [
    { "field": "issue_type", "operator": "eq", "value": "bug" },
    { "field": "priority", "operator": "eq", "value": "p0" }
  ],
  "logic": "AND",
  "notification": {
    "channels": ["站内信", "短信"],
    "recipients": ["项目负责人", "安全管理员"]
  }
}
```

### 3.X 通知汇总/摘要（V2.0）

对于不紧急的更新，支持汇总通知：

| 汇总频率 | 说明 |
|---------|------|
| 实时 | 即时发送 |
| 每小时 | 每小时汇总一次 |
| 每天 | 每天早上 9 点发送昨日汇总 |
| 每周 | 每周一早上发送上周汇总 |

**汇总邮件内容：**
- 我负责的任务状态变更（X 条）
- 我关注的 Issue 有新评论（X 条）
- 我的工时记录提醒（X 条待填写）
- 项目动态（X 条）

### 3.X 通知偏好中心（V2.0）

用户可按模块和事件类型精细化配置通知接收方式：

| 通知类型 | 站内信 | 邮件 | 钉钉 | 企业微信 | 飞书 |
|---------|-------|------|------|---------|------|
| 任务分配给我 | ✅ | ✅ | ✅ | ✅ | ✅ |
| 任务评论 @我 | ✅ | ✅ | ✅ | ✅ | ✅ |

| 迭代开始/结束 | ✅ | ✅ | ❌ | ❌ | ❌ |

# 任务管理 PRD — Core V1.0

> **Jira 映射**：Jira Issue、Backlog、Sprint、Board 的基础闭环。
>
> **开发口径**：V1.0 只做固定模型的 Issue 管理，不做自定义字段、工作流方案、Worklog、批量操作、回收站、导入导出、代码仓库、CI/CD、WakaTime、会议、AI 或插件运行时。
>
> **关联任务单**：[任务管理_tasks.md](../../../2-tasks/core/V1.0/任务管理_tasks.md)

## 1. V1.0 范围

| 模块 | V1.0 必须交付 | 小版本 |
|------|---------------|--------|
| Issue 基础 | Issue Key、固定类型、固定状态、CRUD、详情、列表筛选 | 1.3 |
| 协作 | 评论、@ 提及、关注、标签、活动历史 | 1.3 |
| 结构 | 子任务、任务依赖、阻塞校验 | 1.3 |
| 敏捷 | Backlog、Sprint、Board、拖拽、基础筛选 | 1.4 |
| 附件 | 上传、下载、删除、元数据、权限校验 | 1.5 |
| 通知事件 | task.created、task.assigned、task.commented、task.status_changed、task.due_soon、task.overdue | 1.5 |

## 2. 数据模型边界

| 对象 | V1.0 设计 |
|------|-----------|
| Issue Key | `{PROJECT_KEY}-{SEQ}`，项目内自增，创建后不可变 |
| Issue 类型 | Epic / Story / Task / Bug / Subtask，固定枚举 |
| Issue 状态 | To Do / In Progress / Done，固定状态流 |
| 优先级 | P0 / P1 / P2 / P3 |
| 预估 | 保留 Story Point 和预计工时字段，只做展示和筛选 |
| 标签 | 项目级标签，支持颜色 |
| 子任务 | 最多 3 层，子任务继承项目与 Sprint |
| 依赖 | blocks / blocked by，禁止循环依赖 |

## 3. 用户故事

### US-ISS-001 创建 Issue

**作为** 项目成员，**我希望** 创建 Issue，**以便** 把工作放入项目。

验收标准：
- [ ] 必填：标题、项目、类型
- [ ] 可选：描述、负责人、优先级、截止日期、标签、Sprint、Story Point、预计工时
- [ ] 创建后自动生成 Issue Key
- [ ] 创建者自动关注该 Issue
- [ ] 创建成功后触发 `task.created` 通知事件

### US-ISS-002 查看和编辑 Issue

验收标准：
- [ ] 详情页包含内容区、属性区、评论区、活动历史
- [ ] 标题、描述、负责人、优先级、截止日期、标签可编辑
- [ ] 每次字段变更写入活动历史
- [ ] Viewer 只能查看，Member 可编辑，Admin 可管理

### US-ISS-003 查询 Issue 列表

验收标准：
- [ ] 支持项目、Sprint、负责人、状态、类型、优先级、标签筛选
- [ ] 支持标题/描述关键词搜索
- [ ] 支持分页和排序
- [ ] 普通用户只能看到自己有权访问的项目 Issue

### US-ISS-004 状态流转

验收标准：
- [ ] 支持 To Do -> In Progress -> Done
- [ ] Done 可重新打开到 In Progress
- [ ] 被未完成前置任务阻塞的 Issue 不能进入 Done
- [ ] 状态变更触发活动历史和 `task.status_changed`

### US-COL-001 评论与 @ 提及

验收标准：
- [ ] 支持 Markdown 评论
- [ ] 支持 15 分钟内编辑自己的评论
- [ ] 作者或项目管理员可删除评论
- [ ] @ 提及项目成员时触发通知

### US-COL-002 关注 Watch

验收标准：
- [ ] 用户可关注/取消关注 Issue
- [ ] 创建者、负责人、评论者自动关注
- [ ] 关注者在评论和状态变更时收到通知

### US-COL-003 标签

验收标准：
- [ ] 项目管理员可创建、编辑、删除标签
- [ ] 标签包含名称和颜色
- [ ] Issue 可关联多个标签
- [ ] 列表和 Board 支持按标签筛选

### US-REL-001 子任务

验收标准：
- [ ] 可在 Issue 详情中创建子任务
- [ ] 子任务继承父任务项目和 Sprint
- [ ] 最多 3 层子任务
- [ ] 父任务显示子任务完成数量和进度

### US-REL-002 任务依赖

验收标准：
- [ ] 支持添加 blocks / blocked by 关系
- [ ] 禁止循环依赖
- [ ] 被阻塞 Issue 不能进入 Done
- [ ] 前置任务完成后触发 `task.dependency_unblocked`

### US-AG-001 Backlog

验收标准：
- [ ] 显示未规划 Issue
- [ ] 支持拖入/移出 Sprint
- [ ] 支持 Backlog 内排序
- [ ] 支持按类型、优先级、负责人、标签筛选

### US-AG-002 Sprint

验收标准：
- [ ] 支持创建、编辑、启动、完成 Sprint
- [ ] 启动后记录实际开始时间
- [ ] 完成 Sprint 时可选择未完成 Issue 迁移到 Backlog 或下一个 Sprint
- [ ] Sprint 启动/完成触发通知事件

### US-AG-003 Board

验收标准：
- [ ] 默认三列：To Do / In Progress / Done
- [ ] 支持跨列拖拽改变状态
- [ ] 支持列内排序
- [ ] 拖拽失败时回滚
- [ ] 支持“我负责的、未完成、优先级、标签、Sprint”筛选

### US-ATT-001 附件

验收标准：
- [ ] 支持上传、下载、删除任务附件
- [ ] 附件显示文件名、大小、类型、上传人、上传时间
- [ ] 删除附件写入活动历史
- [ ] 附件访问遵守项目权限

## 4. 权限规则

| 操作 | Admin | Member | Viewer |
|------|-------|--------|--------|
| 查看 Issue | 是 | 是 | 是 |
| 创建 Issue | 是 | 是 | 否 |
| 编辑 Issue | 是 | 是 | 否 |
| 删除 Issue | 是 | 创建者可删 | 否 |
| 管理标签 | 是 | 否 | 否 |
| 评论 | 是 | 是 | 否 |
| 上传附件 | 是 | 是 | 否 |
| 调整 Sprint / Board | 是 | 是 | 否 |

平台管理员拥有项目管理员等效权限，但所有越权操作必须写入审计日志。

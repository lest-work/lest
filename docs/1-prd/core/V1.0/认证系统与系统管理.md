# 认证系统与平台管理 PRD — Core V1.0

> **Jira 映射**：Atlassian Account 基础登录 + Jira System Admin 的最小平台设置。
>
> **开发口径**：V1.0 只做账号、登录、Token、个人资料、平台管理员、项目邀请和基础安全策略。不做 OA 用户管理、部门、岗位、菜单、字典、SSO、LDAP、SCIM、多租户或插件管理。

## 1. V1.0 范围

| 模块 | V1.0 必须交付 | 小版本 |
|------|---------------|--------|
| 初始化 | 首个平台管理员创建、至少保留 1 名平台管理员 | 1.1 |
| 登录 | 邮箱/用户名 + 密码登录、Access Token、Refresh Token、退出 | 1.1 |
| 个人资料 | 昵称、头像、邮箱、密码修改 | 1.1 / 1.2 |
| 项目邀请 | 邮箱邀请、邀请链接注册或登录后加入项目 | 1.2 |
| 平台管理员 | 管理邮件、安全、审计配置；可设定其他平台管理员 | 1.2 |
| 安全基础 | 密码策略、登录失败锁定、退出所有设备 | 1.2 |
| 审计 | 登录、登出、密码修改、平台设置、成员变更记录 | 1.2 |

## 2. 用户故事

### US-AUTH-001 首次初始化

验收标准：
- [ ] 首次启动时进入初始化页面
- [ ] 输入邮箱和密码创建首个平台管理员
- [ ] 初始化完成后不可重复初始化
- [ ] 系统始终至少保留 1 名平台管理员

### US-AUTH-002 登录和刷新 Token

验收标准：
- [ ] 支持邮箱或用户名 + 密码登录
- [ ] 登录成功返回 Access Token 和 Refresh Token
- [ ] Access Token 过期可通过 Refresh Token 刷新
- [ ] 退出登录后 Refresh Token 失效
- [ ] 密码错误提示不暴露账号是否存在

### US-AUTH-003 个人资料和密码

验收标准：
- [ ] 用户可修改昵称、头像、邮箱
- [ ] 修改密码必须输入当前密码
- [ ] 修改密码后所有 Refresh Token 失效
- [ ] 个人资料变更写入活动记录

### US-AUTH-004 项目邀请注册

验收标准：
- [ ] 项目 Admin 可输入邮箱邀请用户
- [ ] 邀请邮件包含一次性链接，默认 7 天有效
- [ ] 已有账号点击链接后直接加入项目
- [ ] 新用户点击链接后先注册，再自动加入项目
- [ ] 邀请状态可查看：待接受、已接受、已过期、已撤销

### US-ADMIN-001 平台管理员

验收标准：
- [ ] 平台管理员可进入平台设置
- [ ] 平台管理员可设置或取消其他平台管理员
- [ ] 禁止取消最后一个平台管理员
- [ ] 平台管理员可跨项目排障，但越权操作必须审计

### US-SEC-001 安全策略

验收标准：
- [ ] 可配置密码最小长度和复杂度
- [ ] 连续失败登录达到阈值后锁定账号
- [ ] 用户可退出所有设备
- [ ] 平台管理员可查看登录失败日志

## 3. 权限模型

| 层级 | 角色 | 权限 |
|------|------|------|
| 平台级 | platform_admin | 平台设置、审计查询、管理员授权、跨项目排障 |
| 项目级 | Admin | 项目设置、成员、迭代、里程碑、Issue 管理 |
| 项目级 | Member | 创建和编辑 Issue、评论、附件、Sprint/Board 操作 |
| 项目级 | Viewer | 只读查看项目、Issue、Board |

## 4. 删除 OA 能力

| 不做项 | 原因 |
|--------|------|
| 部门管理 | 敏捷平台按项目组织，不按行政部门组织 |
| 岗位管理 | 岗位是 HR 概念，不是 Jira Core 能力 |
| 菜单管理 | 菜单由产品固定，按角色展示 |
| 字典管理 | 枚举和字段配置进入 V2.0 Scheme |
| 管理员录入用户 | V1.0 采用注册和邀请，企业目录进入 V5.0 |

## 5. V1.0 不做

| 功能 | 归属 |
|------|------|
| SSO、LDAP、SCIM、企业 IdP | V5.0 商业版 |
| 多租户 | V5.0 商业版 |
| 插件管理、插件权限、插件配置 | V4.0 插件平台 |
| 短信登录、社交登录 | 后续插件或商业增强 |

# 系统设置 PRD
> **📌 Jira 映射**：Jira System Administration（Jira 没有独立的"系统设置"菜单，功能分散在各处的 Admin 入口）
>
> **功能定位**：平台级配置（邮件、安全、存储、审计），仅供平台管理员访问。普通用户看不到这个菜单。

> **安全增强**：2FA/MFA、API Token、可信网络、防暴力破解等安全功能详见 [安全增强.md](./安全增强.md)。

## 1. 核心设计理念

### 1.1 LEST 不需要什么

> 以下是传统管理系统标配、但敏捷平台不需要的功能：

| 删除项 | 删除理由 |
|--------|---------|
| 字典管理 | 敏捷平台的数据枚举用自定义字段（EAV）管理，不是字典 |
| 公告管理 | 通过通知系统推送消息，不需要固定公告 |
| 部门管理 | 见认证系统 PRD，用户不按部门组织 |
| 岗位管理 | 同上 |
| 用户管理 | 见认证系统 PRD，用户通过邀请加入 |
| 菜单管理 | 功能菜单由前端固定定义，按角色展示 |
| 定时任务管理 | 定时任务（Job）不是平台配置，是后端基础设施 |

### 1.2 LEST 平台真正需要的配置

| 类别 | 配置项 | 说明 |
|------|--------|------|
| **邮件** | SMTP 服务器 | 发送通知邮件 |
| **安全** | 密码策略 | 最小长度、复杂度 |
| **安全** | 会话超时 | Access Token 有效期 |
| **安全** | 登录限制 | 登录失败锁定次数 |
| **存储** | 文件存储 | 本地 / OSS（可选）|
| **通知** | 通知渠道开关 | 站内信、邮件、IM |
| **审计** | 审计日志 | 平台级操作记录 |

## 2. 用户故事

### 2.1 平台管理员访问

#### US-001: 进入平台设置
**作为** 平台管理员，**我希望** 在专用入口访问平台设置，**以便** 配置平台级参数。

验收标准：
- [ ] 平台设置入口在页面右上角齿轮图标
- [ ] 仅 `platform_admin` 角色可见
- [ ] 非平台管理员点击齿轮后只能进入"个人设置"

### 2.2 邮件配置

#### US-002: 配置邮件服务
**作为** 平台管理员，**我希望** 配置 SMTP 服务器，**以便** 让平台发送通知邮件。

验收标准：
- [ ] 配置项：SMTP 地址、端口、账号、密码、是否使用 SSL
- [ ] 可发送测试邮件验证配置
- [ ] 配置保存后即时生效

### 2.3 安全设置

#### US-003: 配置密码策略
**作为** 平台管理员，**我希望** 配置密码策略，**以便** 提高平台安全性。

验收标准：
- [ ] 最小密码长度（默认 8 位）
- [ ] 是否要求包含特殊字符（默认否）
- [ ] 密码修改后强制失效所有会话（可选）

#### US-004: 配置登录安全
**作为** 平台管理员，**我希望** 配置登录安全策略，**以便** 防止暴力破解。

验收标准：
- [ ] 登录失败锁定次数（默认 5 次）
- [ ] 锁定时长（默认 15 分钟）
- [ ] Session 超时时间（默认 30 分钟）

### 2.4 审计日志

#### US-005: 查看审计日志
**作为** 平台管理员，**我希望** 查看平台级操作日志，**以便** 审计安全事件。

验收标准：
- [ ] 显示：时间操作用户、操作类型、目标、IP
- [ ] 支持按操作类型（登录/登出/配置变更等）、时间范围筛选
- [ ] 支持分页
- [ ] 日志不可删除，仅可查询
- [ ] 默认保留 90 天

## 4. 页面原型（Jira 思路）

### 4.1 平台设置入口

> **设计说明**：Jira 没有"系统设置"这个独立菜单。LEST 参照 Jira，将平台设置放在右上角入口，但仅对平台管理员可见。

```
┌────────────────────────────────────────────────────────────┐
│  LEST Platform           [搜索...]     🔔   👤 张三 ▼   [⚙]  │
└────────────────────────────────────────────────────────────┘
                                                              ↑
                                                    仅 Admin 可见
                                                    点击进入平台设置
```

### 4.2 平台设置页

```
┌────────────────────────────────────────────────────────────┐
│  平台设置                                         [×]        │
├────────────────────────────────────────────────────────────┤
│  [邮件] [安全] [审计日志]                                       │
├────────────────────────────────────────────────────────────┤
│  邮件配置                                                    │
│  ────────────────────────────────────────────────────────── │
│  SMTP 地址:   [smtp.example.com                         ]   │
│  端口:       [465                                     ]   │
│  用户名:     [noreply@example.com                      ]   │
│  密码:       [●●●●●●●●●●●●●                           ]   │
│  使用 SSL:   [● 是 ○ 否]                                   │
│  发件人名称: [LEST Platform                            ]   │
│  发送测试:   [test@example.com  ] [发送测试邮件]              │
│                                                              │
│                                        [取消]  [保存配置]     │
└────────────────────────────────────────────────────────────┘
```

## 5. 与 V1.0 PRD 的差异

| 差异项 | V1.0 | V2.0 |
|--------|------|------|
| 字典管理 | ✅ 有（CRUD）| ❌ 删除（用自定义字段代替）|
| 公告管理 | ✅ 有 | ❌ 删除（用通知系统代替）|
| 部门管理 | ✅ 有（树形）| ❌ 删除（见认证系统 PRD）|
| 岗位管理 | ✅ 有 | ❌ 删除 |
| 用户管理 | ✅ 有（管理员录入）| ❌ 删除（见认证系统 PRD）|
| 菜单管理 | ✅ 有（菜单树 CRUD）| ❌ 删除（菜单固定）|
| 定时任务 | ✅ 有 | ❌ 删除（后端基础设施）|
| 系统监控 | ✅ 有（磁盘/CPU/Redis）| ❌ 删除（用外部监控工具）|
| 在线用户 | ✅ 有 | ❌ 删除（非核心需求）|
| 数据备份 | ✅ 有 | ❌ 删除（运维层面处理）|
| 邮件配置 | ✅ 有 | ✅ 保留并完善 |
| 安全策略 | ✅ 有 | ✅ 保留并精简 |
| 审计日志 | ✅ 有（记录所有操作）| ✅ 保留（仅记录平台级操作）|

> **数据库设计与 API**：详细数据库表结构设计见 [安全增强.md](./安全增强.md)；详细 API 接口见 [认证系统与系统管理.md](./认证系统与系统管理.md)。本节只列出主要接口路径和表名概览。

## 7. 接口设计（概览）

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | `/platform/config` | 获取平台配置 | platform_admin |
| PUT | `/platform/config` | 更新平台配置 | platform_admin |
| GET | `/platform/config/smtp/test` | 发送测试邮件 | platform_admin |
| GET | `/audit-log` | 查询审计日志 | platform_admin |
| POST | `/auth/mfa/setup` | 启用 2FA | 登录用户 |
| POST | `/auth/mfa/verify` | 验证 2FA | 登录用户 |
| POST | `/auth/pat` | 创建 PAT | 登录用户 |
| GET | `/auth/pat` | 获取 PAT 列表 | 登录用户 |
| DELETE | `/auth/pat/{id}` | 撤销 PAT | 登录用户 |

> 详细接口设计见 [认证系统与系统管理.md](./认证系统与系统管理.md)

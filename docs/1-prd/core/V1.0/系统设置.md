# 系统设置 PRD — Core V1.0

> **Jira 映射**：Jira System Administration 的最小平台设置。
>
> **开发口径**：V1.0 系统设置只保留平台运行必需配置：邮件、安全、审计、附件存储基础。OA 配置、监控、定时任务、数据备份、插件管理和企业身份配置都不进入 V1.0。

## 1. V1.0 范围

| 模块 | V1.0 必须交付 | 小版本 |
|------|---------------|--------|
| 邮件配置 | SMTP 地址、端口、账号、密码、TLS/SSL、测试发送 | 1.2 / 1.5 |
| 安全策略 | 密码最小长度、复杂度、登录失败锁定、会话超时 | 1.2 |
| 审计日志 | 查询平台级和核心业务写操作日志 | 1.2 |
| 附件存储 | 本地存储配置、大小限制 | 1.5 |

## 2. 用户故事

### US-SET-001 进入平台设置

验收标准：
- [ ] 只有 platform_admin 可见平台设置入口
- [ ] 非管理员只能进入个人设置
- [ ] 平台设置包含邮件、安全、审计、附件存储四类

### US-SET-002 配置 SMTP

验收标准：
- [ ] 可配置 SMTP 地址、端口、账号、密码、TLS/SSL
- [ ] 可发送测试邮件
- [ ] 配置变更写入审计日志

### US-SET-003 配置安全策略

验收标准：
- [ ] 可设置密码最小长度和复杂度
- [ ] 可设置登录失败锁定次数和锁定时长
- [ ] 可设置 Access Token 与 Refresh Token 有效期

### US-SET-004 查看审计日志

验收标准：
- [ ] 显示时间、操作者、操作、资源、IP
- [ ] 支持按用户、时间、操作类型筛选
- [ ] 日志不可删除
- [ ] 默认保留 90 天

## 3. V1.0 不做

| 功能 | 归属 |
|------|------|
| 部门、岗位、菜单、字典、公告 | 删除，不属于 Jira-like Core |
| 系统监控、定时任务、备份恢复 | 运维体系或 V5.0 企业版 |
| 插件管理 | V4.0 插件平台 |
| SSO、LDAP、SCIM、企业 IdP | V5.0 商业版 |

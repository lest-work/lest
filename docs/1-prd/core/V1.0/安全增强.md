# 安全增强 PRD — Core V1.0

> **定位**：V1.0 只做 Core 运行所必需的账号安全和 API 访问安全。高级身份安全进入 V5.0 企业商业版。

## 1. V1.0 范围

| 能力 | V1.0 设计 | 小版本 |
|------|-----------|--------|
| 密码策略 | 最小长度、复杂度、修改密码后撤销会话 | 1.2 |
| 登录失败锁定 | 连续失败次数、锁定时长、失败日志 | 1.2 |
| 退出所有设备 | 用户主动撤销全部 Refresh Token | 1.2 |
| API Key | 创建、撤销、过期时间、权限范围、最后使用时间 | 1.5 |
| Webhook 签名 | HMAC-SHA256 签名和重试基础 | 1.5 |
| 审计日志 | 登录、配置、项目成员、Issue 写操作审计 | 1.2 / 1.6 |

## 2. 用户故事

### US-SEC-001 密码策略

验收标准：
- [ ] 平台管理员可配置密码最小长度
- [ ] 可配置是否要求数字、大小写字母、特殊字符
- [ ] 用户修改密码后所有会话失效

### US-SEC-002 登录失败锁定

验收标准：
- [ ] 连续失败达到阈值后锁定账号
- [ ] 锁定期间登录返回统一错误
- [ ] 登录失败日志记录 IP、UserAgent、时间、失败原因

### US-SEC-003 API Key

验收标准：
- [ ] 用户可创建和撤销自己的 API Key
- [ ] API Key 只在创建时展示一次
- [ ] 支持过期时间和权限范围
- [ ] API Key 调用写入最后使用时间

### US-SEC-004 Webhook 签名

验收标准：
- [ ] 出站 Webhook 使用 HMAC-SHA256 签名
- [ ] 支持重试次数和失败日志
- [ ] Webhook 失败不阻塞核心业务操作

## 3. V1.0 不做

| 功能 | 归属 |
|------|------|
| TOTP 2FA、FIDO2、短信验证码 | V5.0 企业商业版或安全插件 |
| SSO、LDAP、SCIM | V5.0 企业商业版 |
| 可信网络、设备信任、条件访问 | V5.0 企业商业版 |

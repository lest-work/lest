## Core 定位说明

Core 是 LEST Platform 的基础层，设计原则：

- **最小化**：只包含平台运行所必需的功能
- **稳定性**：Core 模块变更频率低，API 稳定
- **扩展性**：所有可插拔功能都通过插件系统扩展
- **开源性**：Core 全部开源（MIT License）

## V1.0 → V2.0 架构升级

### 认证与平台管理（重大修订）

> **V2.0 说明**：V1.0 的认证系统 PRD 最初参考了传统 OA 管理系统，引入了不适用于敏捷平台的功能（部门树、岗位管理、字典管理、菜单管理、用户 CRUD）。V2.0 重写后，以 Jira 为标杆，删除了所有不相关功能。

| V1.0（删除）| V2.0（保留/新增）|
|------------|----------------|
| 用户 CRUD（管理员录入）| 用户邀请制 + SSO（V3.0）|
| 部门管理（树形）| 删除，按项目成员管理 |
| 岗位管理 | 删除 |
| 字典管理 | 删除，用 EAV 自定义字段代替 |
| 菜单管理（数据库配置）| 删除，菜单固定 |
| 全局角色-菜单权限 | 删除，用项目级角色代替 |

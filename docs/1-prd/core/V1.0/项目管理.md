# 项目管理 PRD — Core V1.0

> **Jira 映射**：Jira Project、Project Roles、Sprint、Milestone/Version 的基础能力。
>
> **开发口径**：V1.0 项目是 Core 顶级容器，只包含项目、成员、角色、迭代、里程碑和项目级权限。代码仓库、CI/CD、会议、WakaTime、IM、审批和组织同步全部是插件或企业能力。

## 1. V1.0 范围

| 模块 | V1.0 必须交付 | 小版本 |
|------|---------------|--------|
| 项目 | 创建、编辑、归档、恢复、删除保护、项目 Key | 1.2 |
| 成员 | 添加、移除、邮箱邀请、角色调整、最后管理员保护 | 1.2 |
| 权限 | Admin / Member / Viewer、项目级 API 校验、服务层兜底 | 1.2 |
| 迭代 | 创建、编辑、启动、完成、取消、任务统计 | 1.2 / 1.4 |
| 里程碑 | 创建、编辑、关联迭代、进度展示 | 1.2 |
| 审计 | 项目和成员写操作记录审计日志 | 1.2 |

## 2. 用户故事

### US-PRJ-001 创建项目

验收标准：
- [ ] 输入项目名称、项目 Key、描述、负责人
- [ ] 项目 Key 全局唯一，用于 Issue Key 前缀
- [ ] 创建者自动成为项目 Admin
- [ ] 创建项目写入审计日志

### US-PRJ-002 编辑和归档项目

验收标准：
- [ ] Admin 可编辑名称、描述、负责人
- [ ] 项目 Key 创建后不可修改
- [ ] 归档后项目只读，不出现在活跃列表
- [ ] 可恢复已归档项目

### US-PRJ-003 删除保护

验收标准：
- [ ] V1.0 不做项目回收站
- [ ] 删除项目前必须确认
- [ ] 存在未完成 Sprint 或未完成 Issue 时禁止删除
- [ ] 删除操作写入审计日志

### US-MEM-001 添加项目成员

验收标准：
- [ ] Admin 可通过用户搜索或邮箱邀请添加成员
- [ ] 邀请链接一次性使用，默认 7 天有效
- [ ] 被邀请用户注册或登录后自动加入项目
- [ ] 成员默认角色为 Member

### US-MEM-002 移除成员和调整角色

验收标准：
- [ ] Admin 可移除成员或修改角色
- [ ] 不能移除或降级最后一个 Admin
- [ ] 移除成员后，其负责的未完成 Issue 变为未分配
- [ ] 成员变更触发通知和审计日志

### US-ITER-001 管理迭代

验收标准：
- [ ] Admin 可创建、编辑、启动、完成、取消迭代
- [ ] 迭代包含名称、目标、开始日期、结束日期、状态
- [ ] 同一项目内进行中的 Sprint 只能有一个
- [ ] 完成迭代时可处理未完成 Issue

### US-MILE-001 管理里程碑

验收标准：
- [ ] Admin 可创建、编辑、删除里程碑
- [ ] 里程碑可关联多个迭代
- [ ] 里程碑展示完成进度、剩余天数、未完成 Issue 数

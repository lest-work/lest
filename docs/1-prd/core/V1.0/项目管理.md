## 1. 概述

### 1.1 功能背景

项目管理是 LEST Platform 的核心功能之一，为开发团队提供项目的创建、成员管理、迭代规划和里程碑追踪能力。一个项目可以包含多个迭代（Sprint）和多个里程碑，成员可以参与多个项目。

### 1.2 功能目标

- 提供项目的基本管理能力（创建、编辑、归档、删除）
- 支持多成员协作的项目团队管理
- 提供迭代（Sprint）管理，支持敏捷开发流程
- 支持里程碑管理，跟踪项目关键节点
- 提供项目概览和统计信息

### 1.3 目标用户

- **项目管理员**：创建和管理项目，分配项目成员
- **项目经理**：规划迭代、管理里程碑、跟踪项目进度
- **开发人员**：查看自己参与的项目和迭代
- **审计人员**：查看项目整体情况

## 2. 用户故事

### 2.1 项目管理

#### US-001: 创建项目
**作为** 项目管理员，**我希望** 创建一个新项目，**以便** 在平台上开始一个软件开发项目。

验收标准：
- [ ] 输入项目名称（必填，唯一性校验）、描述、起止时间
- [ ] 设置项目负责人（默认为创建者）
- [ ] 创建者自动成为项目管理员
- [ ] 支持选择项目模板（可选，敏捷/瀑布/看板）

#### US-002: 编辑项目
**作为** 项目管理员，**我希望** 编辑项目信息，**以便** 修正错误或更新项目信息。

验收标准：
- [ ] 可编辑名称（唯一性校验）、描述、起止时间
- [ ] 可更换项目负责人
- [ ] 不可编辑项目创建时间
- [ ] 编辑操作记录到操作日志

#### US-003: 删除项目
**作为** 项目管理员，**我希望** 删除项目，**以便** 清理不需要的项目。

验收标准：
- [ ] 删除前需确认（防止误删）
- [ ] 删除时检查项目下是否有未完成的迭代
- [ ] 若有未完成的迭代，提示用户先归档或删除
- [ ] 删除后数据进入回收站（软删除），30 天后可彻底删除

#### US-004: 归档项目
**作为** 项目管理员，**我希望** 归档已完成的项目，**以便** 将已完成项目从活跃列表移除。

验收标准：
- [ ] 归档后可查看，不可编辑
- [ ] 归档后可取消归档，恢复为活跃状态
- [ ] 归档项目不影响已有数据的关联关系

#### US-005: 查询项目列表
**作为** 平台用户，**我希望** 查看我参与的项目列表，**以便** 快速找到需要管理的项目。

验收标准：
- [ ] 普通用户只看到自己参与的项目
- [ ] 管理员可以看到所有项目
- [ ] 支持按名称模糊搜索
- [ ] 支持按状态筛选（活跃/已归档）
- [ ] 支持按时间范围筛选
- [ ] 支持分页，每页默认 20 条

#### US-006: 项目详情
**作为** 项目成员，**我希望** 查看项目详细信息，**以便** 了解项目整体情况。

验收标准：
- [ ] 显示项目基本信息（名称、描述、负责人、起止时间）
- [ ] 显示项目统计（成员数、迭代数、任务数、已完成任务数）
- [ ] 显示当前迭代进度
- [ ] 显示最近活动时间

### 2.2 项目成员管理

#### US-007: 添加项目成员
**作为** 项目管理员，**我希望** 向项目中添加成员，**以便** 让其他用户参与项目。

验收标准：
- [ ] 支持搜索用户（按用户名/昵称）
- [ ] 支持设置成员角色（管理员/开发者/观察者）
- [ ] 管理员可添加/移除其他管理员和开发者
- [ ] 观察者只能查看，不能添加其他成员
- [ ] 成员列表显示用户名、昵称、角色、加入时间

#### US-008: 移除项目成员
**作为** 项目管理员，**我希望** 从项目中移除成员，**以便** 管理团队组成。

验收标准：
- [ ] 移除前需确认
- [ ] 移除后，该成员的任务保留但变为未分配状态
- [ ] 不能移除自己（最后一个管理员）
- [ ] 移除操作记录到操作日志

#### US-009: 修改成员角色
**作为** 项目管理员，**我希望** 修改成员的权限角色，**以便** 调整团队权限。

验收标准：
- [ ] 可将开发者升级为管理员
- [ ] 可将管理员降级为开发者
- [ ] 不可将最后一个管理员降级

### 2.3 迭代管理

#### US-010: 创建迭代
**作为** 项目管理员，**我希望** 创建迭代，**以便** 规划项目的工作周期。

验收标准：
- [ ] 输入迭代名称（必填）、目标、起止时间
- [ ] 迭代时间不可重叠
- [ ] 创建后迭代状态为"计划中"

#### US-011: 启动迭代
**作为** 项目管理员，**我希望** 启动迭代，**以便** 开始新一轮工作。

验收标准：
- [ ] 启动前需确认
- [ ] 启动后迭代状态变为"进行中"
- [ ] 启动时自动将迭代开始时间更新为当前时间
- [ ] 启动后不可编辑迭代时间

#### US-012: 结束迭代
**作为** 项目管理员，**我希望** 结束迭代，**以便** 标记迭代完成。

验收标准：
- [ ] 结束前可选择是否生成迭代报告
- [ ] 结束后的迭代状态变为"已完成"
- [ ] 结束后的迭代不可编辑
- [ ] 未完成的任务可选择：带入下一个迭代 / 保持不变

#### US-013: 查询迭代列表
**作为** 项目成员，**我希望** 查看项目下的迭代列表，**以便** 了解迭代安排。

验收标准：
- [ ] 显示所有迭代（计划中/进行中/已完成）
- [ ] 显示每个迭代的任务统计
- [ ] 支持按状态筛选
- [ ] 支持按时间排序

### 2.4 里程碑管理

#### US-014: 创建里程碑
**作为** 项目管理员，**我希望** 创建里程碑，**以便** 标记项目的关键节点。

验收标准：
- [ ] 输入里程碑名称（必填）、目标日期、描述
- [ ] 里程碑日期可调整
- - [ ] 可关联多个迭代

#### US-015: 里程碑进度
**作为** 项目成员，**我希望** 查看里程碑的完成进度，**以便** 了解项目是否按计划推进。

验收标准：
- [ ] 显示关联迭代的完成情况
- [ ] 显示未完成任务数量
- [ ] 显示距离目标日期的剩余天数
- [ ] 支持设置提醒（里程碑前 N 天通知）

### 3.2 业务流程

#### 3.2.1 创建项目流程

```
项目管理员点击"新建项目"
          │
          ▼
    输入项目信息
   (名称、描述、时间、模板)
          │
          ▼
    名称唯一性校验
   ┌─────┴─────┐
   │  名称已存在  │────────► 返回错误，提示重新输入
   └───────────┘
   │  名称可用
   ▼
    创建项目记录
   (状态: 活跃)
          │
          ▼
    创建者添加为项目管理员
          │
          ▼
    返回项目详情页
```

#### 3.2.2 迭代管理流程

```
创建迭代
    │
    ▼
设置迭代名称、目标、时间
    │
    ▼
时间冲突检测
   ┌─────┴─────┐
   │  与已有迭代  │────────► 返回错误，提示调整时间
   │  时间重叠    │
   └───────────┘
   │  时间不冲突
   ▼
    迭代创建成功 (状态: 计划中)
    │
    ▼
启动迭代
    │
    ▼
状态变更为"进行中"
    │
    ▼
添加任务到迭代
    │
    ▼
结束迭代
    │
    ├─── 未完成任务带入下一迭代
    │
    ▼
状态变更为"已完成"
```

## 6. 验收标准

### 6.1 项目管理

| 用例 | 验收标准 |
|------|---------|
| UC-001 | 管理员可创建项目，项目名称全局唯一 |
| UC-002 | 管理员可编辑项目信息，编辑操作记录日志 |
| UC-003 | 管理员可删除项目，需确认提示 |
| UC-004 | 删除前检查是否有未完成的迭代 |
| UC-005 | 管理员可归档项目，归档后可恢复 |
| UC-006 | 用户只能看到自己参与的项目 |
| UC-007 | 项目详情显示统计数据 |

### 6.2 项目成员管理

| 用例 | 验收标准 |
|------|---------|
| UC-008 | 管理员可添加项目成员，设置角色 |
| UC-009 | 管理员可移除项目成员，需确认提示 |
| UC-010 | 管理员可修改成员角色 |
| UC-011 | 不可移除最后一个管理员 |
| UC-012 | 被移除成员的未完成任务变为未分配状态 |

### 6.3 迭代管理

| 用例 | 验收标准 |
|------|---------|
| UC-013 | 管理员可创建迭代，设置名称、目标、时间 |
| UC-014 | 迭代时间不可与已有迭代重叠 |
| UC-015 | 管理员可启动迭代，状态变为"进行中" |
| UC-016 | 管理员可结束迭代，可选择未完成任务处理方式 |
| UC-017 | 迭代完成后状态变为"已完成"，不可编辑 |
| UC-018 | 可查看迭代下的任务列表 |

### 6.4 里程碑管理

| 用例 | 验收标准 |
|------|---------|
| UC-019 | 管理员可创建里程碑 |
| UC-020 | 可将迭代关联到里程碑 |
| UC-021 | 里程碑显示关联迭代的完成进度 |

| 7003 | `PROJECT_HAS_MEMBERS` | 409 | 项目下存在成员，无法删除 |

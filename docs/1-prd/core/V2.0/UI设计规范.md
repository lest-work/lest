### 2.3 创建 Bug 弹窗（Create Bug）

**参考**: Jira Create Issue Dialog

**路由**: `/issue/create?type=bug`（模态框）

**所有字段**（按 Screen 布局分组）:

```
┌─────────────────────────────────────────────────────────────────┐
│  🐛 创建 Bug                                                      │  ← 图标随 Issue Type 变化
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  项目 *  [LEST-DEV ▾]                                          │  ← 必填
│                                                                 │
│  Issue Type *  [Bug ▾]                                         │  ← Bug/Story/Task/Epic/子任务
│                                                                 │
│  摘要 *  [                                            ]       │  ← 必填，单行输入
│                                                                 │
│  描述        [富文本编辑器                              ]       │
│              [ 支持 Markdown / @提及 / #引用 ]               │
│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│  其他字段（由 Screen Scheme 决定显示哪些）                        │
│                                                                 │
│  优先级         [P1 - High ▾]                                  │
│  指派给         [@ 选择用户 ▾]                                   │
│  报告人         [@ 自动填充当前用户 ▾]                           │
│  标签           [🏷 + 添加标签 ▾]                                 │
│  史诗           [🎯 选择 Epic ▾]                                 │
│  迭代           [🚀 Sprint 23 ▾]                                │
│  截止日期       [📅 2026-06-10     ]                           │
│  预估工时       [⏱ 8h ]                                        │
│  故事点         [🔢 5 ]                                         │
│  组件           [🧩 组件A ▾]                                      │
│  版本           [🏷 v1.2 ▾]                                     │
│  附件           [📎 拖拽或点击上传]                              │
│  关联 Issue     [+ 添加关联]                                       │
│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│  [取消]                        [创建]  [创建并新建]              │
└─────────────────────────────────────────────────────────────────┘
```

**Bug 专属附加字段**（Bug Issue Type 特有的额外字段）:

| 字段 | 类型 | 说明 | 备注 |
|------|------|------|------|
| 缺陷等级 | Select (致命/严重/一般/轻微) | Bug 严重程度分级 | 默认为"一般" |
| 复现步骤 | Text Area | 复现步骤说明 | 支持 Markdown |
| 环境 | Text Field | 复现环境（浏览器/系统/版本） | 可选 |
| 根因分析 | Text Area | Bug 根因分析 | 可选 |
| 修复版本 | Version Picker | 预计/实际修复版本 | 可选 |
| Bug 类型 | Select (代码错误/需求变更/设计缺陷/测试错误) | Bug 分类 | 可选 |

```
Story 转换规则:
  待办 → 进行中:  条件(指派给!=空) + 校验器(Epic!=空)
  进行中 → 评审中: 条件(故事点!=空)
  评审中 → 已完成: 条件(验收标准已确认)
  已完成 → 重启:   条件(测试发现新问题)
```

#### Task 工作流（4 状态，简化为无评审）

```
┌────────┐      ┌──────────┐      ┌────────┐
│  待办   │ ──→ │  进行中  │ ──→ │ 已完成  │
│  To Do │      │In Progress│     │  Done  │
└────────┘      └──────────┘      └────────┘
    │                 │                │
    │◅────────────────┘                │
    │                  ◅────────────────┘
    │
    │        ┌──────────┐
    │        │   重启   │
    │        │ Reopened │ ────────────────────────◅
    │        └──────────┘
    └─────────────────────────────────────────────
```

#### Epic 工作流（无状态，自动完成）

Epic 没有工作流状态流转。Epic 的"完成"由系统自动判断：当所有关联的 Story 都 Done 时，Epic 自动标记为完成。

---

### 3.2 工作流可定制项

| 可定制项 | 说明 | 示例 |
|---------|------|------|
| **状态** | 可新增/编辑/删除状态 | + "技术设计" / "UAT测试" / "已验收" |
| **转换** | 可新增/编辑/删除状态转换路径 | 从"评审中"→"待办"添加退回路径 |
| **条件** | 转换时的准入条件 | 指派给!=空 / 描述!=空 / 优先级=P1 |
| **校验器** | 转换时的数据校验 | 必填字段检查 / 自定义脚本 |
| **动作** | 转换后的后置处理 | 自动分配 / 发通知 / 创建子任务 / 发 Webhook |
| **转换屏幕** | 每个转换可挂专属屏幕 | 从"待办"→"进行中"弹出"开始工作"表单 |
| **全局转换** | 从任何状态都可到达的转换 | "阻塞" / "取消" |

| 模板 | 状态数 | 适用场景 |

### 5.2 字段上下文（Field Context）

每个自定义字段可设置**生效范围**:

| 上下文类型 | 说明 | 优先级 |
|-----------|------|--------|
| **全局** | 所有项目和所有 Issue Type 都显示 | 最低 |
| **按项目** | 指定项目显示 | 中 |
| **按 Issue Type** | 指定类型显示 | 高 |
| **按项目和类型组合** | 最精确 | 最高 |

示例:
- `缺陷等级` → 仅 Bug 显示（按 Issue Type 上下文）
- `故事点` → 所有项目 + Story/Task 显示（按类型上下文）
- `Epic 进度` → 仅 Epic 显示（按 Issue Type 上下文）
- `代码仓库` → 仅 LEST-DEV 项目显示（按项目上下文）

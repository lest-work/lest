# LEST Platform — UI 设计规范与产品定位

> **版本**: v2.0（重构版）
>
> **日期**: 2026-06-02
>
> **参考系统**: Linear + Jira + Notion
>
> **核心原则**：LEST 不是管理后台，是**敏捷团队的工作台**。用户 80% 的时间在任务看板、Backlog、任务详情页这三个核心场景里，不应该有任何"后台配置"的分心。

---

## 一、产品定位与视觉语言

### 1.1 我们是什么（What We Are）

| 维度 | LEST | 对比 |
|------|------|------|
| 定位 | 敏捷团队工作台 | 不是 OA/ERP/管理后台 |
| 核心交互 | 键盘驱动 + 看板拖拽 | 不是表单配置 |
| 导航深度 | 项目内设置（≤2层）| 不做全局"系统管理"菜单 |
| 视觉重心 | 任务卡片、对话、状态流转 | 不是配置表格 |
| 快捷操作 | 全局 Cmd+K 命令面板 | 不是逐级点击 |

**对标产品**：
- **Linear**：视觉标杆，极简、键盘优先、现代感
- **Jira**：功能深度标杆，完整的敏捷工作流
- **Notion**：编辑体验标杆，任务描述的富文本编辑
- **Figma**：协作标杆，实时多人协同

### 1.2 我们不是什么（What We Are NOT）

- ❌ 不是「系统管理后台」—— 不做字典管理、部门管理、菜单管理
- ❌ 不是「企业 OA」—— 不做考勤、审批流（基础版）、公告
- ❌ 不是「项目管理系统」—— 不做资源管理、甘特图（WBS）
- ❌ 不是「运维平台」—— 不做主机监控、工单系统
- ❌ 不是「文档管理」—— 不做 Wiki、文件库

**核心边界**：只做敏捷项目管理（Issue + Sprint + Kanban），不做一切看起来像"后台"的功能。

## 二、导航架构（消除管理后台感）

### 2.1 全局导航原则

**规则 1：侧边栏只显示工作区，不显示管理区**

```
错误 ❌（管理后台风格）:
LEST Logo
├── 📥 收件箱
├── ✅ 我的任务
├── 📊 仪表盘
├── 📁 项目
│   ├── LEST-DEV
│   └── 移动端 App
├── ⚙️ 系统管理      ← 这是管理后台的标志，应该删除
└── 👤 个人设置

正确 ✅（工作台风格）:
LEST Logo
├── 📥 收件箱
├── ✅ 我的任务
├── 📊 仪表盘
└── 📁 项目
    ├── LEST-DEV        ← 点击展开，进入项目内导航
    └── 移动端 App
```

**规则 2：系统设置入口放在项目内部，不做全局菜单**

- 全局只有 Admin 可见 `/admin`（极简，只包含：用户管理、平台配置、审计日志）
- 普通用户：设置都在「项目 > 项目设置」内
- 个人设置：头像下拉菜单中（不是侧边栏菜单项）

### 2.2 侧边栏（Sidebar）

```
┌────────────────────┐
│ LEST    [−][□][×] │
├────────────────────┤
│ 📥 收件箱 (3)      │  ← 待办通知，类 Linear Inbox
│ ✅ 我的任务 (8)     │  ← 分配给我的任务
│ 📊 仪表盘          │  ← 统计概览
│                    │
│ 📁 项目 ──────── │
│ ├ LEST-DEV         │
│ ├ 移动端 App        │
│ └ + 新建项目        │
│                    │
├────────────────────┤
│ 👤 刘明  [头像▾]   │  ← 头像下拉：个人设置、切换项目、登出
└────────────────────┘
```

**侧边栏交互**：
- 可折叠到图标模式（只显示 Logo + 项目列表）
- 宽度：220px（可拖拽调整）
- 收件箱是核心入口——所有需要处理的事项汇聚于此

### 2.3 顶部栏（TopBar）

```
┌──────────────────────────────────────────────────────────────────────────────────┐
│ [← 返回]  LEST-DEV / 看板         [🔍 筛选 ▾]  [👤 成员]  [⚙ 看板设置]      │
│                                       [⌘K 搜索]         [🔔 3]  [👤 头像]      │
└──────────────────────────────────────────────────────────────────────────────────┘
```

**顶部栏元素**：
- 左：面包屑导航（可点击返回）
- 中：搜索入口（Cmd+K，全局命令面板）
- 右：筛选器、成员头像组、通知（红点）、个人头像

## 三、核心页面设计（工作台风格）

### 3.1 登录页（Login）

**参考**: Linear 登录页

```
原则：极简。不要验证码框、不要记住密码复选框、不要太多字段。

┌────────────────────────────────────────────────────────────┐
│                                                            │
│                                                            │
│                         LEST                              │
│                  敏捷团队工作台                             │
│                                                            │
│            ┌────────────────────────────┐                │
│            │  you@company.com             │ ← 邮箱输入，聚焦即开始     │
│            └────────────────────────────┘                │
│                                                            │
│            ┌────────────────────────────┐                │
│            │  ●●●●●●●●                  │ ← 密码，按 Enter 登录   │
│            └────────────────────────────┘                │
│                                                            │
│                 [登录]                                    │
│                                                            │
│            还没有账号？[联系管理员]                          │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

**设计要点**：
- 全屏居中，无侧边图（避免增加认知负担）
- 验证码通过 `CAPTCHA_PROVIDER` 扩展点静默加载，不显示在登录页（除非检测到异常登录）
- V2.0 Core 登录页只显示账号密码登录；企业微信/飞书/SSO 登录由 V5.0 企业身份能力提供，不进入 V2.0 Core 原型
- 不显示"忘记密码"，通过管理员流程处理

## 五、交互模式规范

### 5.1 不要这样设计（后台风格特征）

| ❌ 后台风格 | ✅ 工作台风格 |
|-----------|------------|
| 点击「系统管理」菜单进入后台 | 设置在项目内或个人资料内 |
| 列表页面用 DataTable（带分页条）| 无限滚动列表（类 Linear/Gmail）|
| 表单提交后弹 Toast 提示「保存成功」| 自动保存（类 Notion），无成功提示 |
| 所有表单字段带 label 标签 | 内联编辑，没有 label（如 Linear） |
| 表格行hover 显示操作按钮 | 卡片右上角有更多操作 `⋮` |
| Tab 切换不同信息区 | 信息区全部展开，不需要 Tab |
| 搜索框在顶部固定 | 搜索框用 ⌘K 命令面板代替 |
| 模态框配置系统参数 | 设置用侧滑抽屉，不打断工作流 |

### 5.2 要这样设计（工作台特征）

**自动保存**：所有编辑无「保存」按钮，Blur 时自动保存，有 Unsaved Indicator（点表示）

**上下文操作**：鼠标 hover 任务卡片才显示操作按钮，节省视觉空间

**键盘优先**：每个核心操作都有快捷键，⌘K 是全局入口

**渐进式披露**：不一次展示所有字段，用户需要时才展开（如 Epic Link）

**空状态友好**：每个空状态都有插画 + 引导文案，不只是「暂无数据」

**实时协作**：类 Figma 的多人协同光标和实时编辑

## 七、与 Jira 的 UI 对照

| 维度 | Jira Cloud | LEST（目标） |
|------|-----------|------------|
| 首页 | Jira 首页（混合信息）| **收件箱**（类 Linear）|

## 1. Core 与 Plugin 边界

### 1.1 Core 是什么

Core 是没有任何插件也必须能运行的 Jira-like 主系统。Core 负责项目协作、任务流转、敏捷迭代、权限、通知、附件、搜索、报表、配置体系和迁移能力。

| 类型 | Core 范围 |
|------|-----------|
| 账号与认证 | 登录、登出、Token、个人资料、密码、平台管理员 |
| 项目 | 项目 CRUD、归档、项目成员、项目角色、迭代、里程碑 |
| 任务 Issue | Epic / Story / Task / Bug / Subtask、任务 CRUD、详情、标签、评论、关注、子任务、依赖、历史 |
| 敏捷 | Backlog、Sprint、Board、拖拽、基础筛选 |
| 通知 | 站内信、WebSocket、邮件 SMTP、通知已读 |
| 附件 | 上传、下载、删除、文件元数据 |
| 权限与审计 | 项目级角色校验、平台管理员绕过、基础审计日志 |
| 系统设置 | 邮件、安全、审计，不包含 OA 部门/岗位/菜单/字典 |
| 开放 API 基础 | API Key、出站 Webhook、基础限流、OpenAPI 文档 |
| 扩展契约 | 事件命名、扩展点接口、前端插槽占位，不包含插件运行时 |

### 1.2 Plugin 是什么

Plugin 是可插拔能力。没有这些插件，Core 仍然必须完整可用。

| 插件/企业能力 | 归属 | 开始开发时间 |
|----------------|------|--------------|
| 会议 | V4.0 官方插件池 | 2027-09-06 之后 |
| 飞书/钉钉/企业微信通知 | V4.0 官方插件池 | 2027-09-06 之后 |
| 飞书/钉钉/企业微信组织同步 | V5.0 企业同步能力 | 2028-03-17 之后 |
| WakaTime/编码追踪 | V4.0 官方插件池 | 2027-09-06 之后 |
| CI/CD | V4.0 官方插件池 | 2027-09-06 之后 |
| 代码仓库集成 | V4.0 官方插件池 | 2027-09-06 之后 |
| AI Provider / AI 助手 | V4.0 AI 插件能力 | 2027-09-06 之后 |
| 团队绩效/高级报表 | V4.0 插件增强 / V5.0 企业报表 | 2027-09-06 之后 |
| SSO / LDAP / SCIM | V5.0 企业商业版 | 2028-03-17 之后 |
| 多租户 | V5.0 企业商业版 | 2028-03-17 之后 |
| OKR / Portfolio / 审批流 | V5.0 企业商业版或 V4 商业插件 | 2028-03-17 之后 |
| 插件市场 / 完整 SDK / 热加载 | V4.0 插件平台 | 2027-09-06 之后 |

> **重要边界**：V1.0/V2.0/V3.0 可以预留扩展点接口，但不能开发插件运行时、插件市场、第三方 SDK 接入或官方插件。

## 2. V1.0 Core 确认范围

**V1.0 时间**：2026-06-04 ~ 2026-08-28  
**目标**：让小团队能真实使用 LEST 管项目、管任务、跑看板和 Sprint。  
**不做插件**：会议、飞书、钉钉、企业微信、WakaTime、CI/CD、代码仓库、AI、绩效都不进入 V1.0。

| 任务 | 固定类型、固定状态、CRUD、详情、标签、评论、关注、子任务、依赖、历史 | [任务管理](./1-prd/core/V1.0/任务管理.md) |

### 2.2 V1.0 明确不开发

| 不开发项 | 原因 | 后续版本 |
|----------|------|----------|
| 会议 | 不是 Jira Core 必需能力 | V4.0 插件 |
| 飞书/钉钉/企业微信 | 第三方 IM 集成，必须插件化 | V4.0 插件；组织同步 V5.0 |
| WakaTime | IDE/编码追踪集成，必须插件化 | V4.0 插件 |
| CI/CD | 外部流水线集成，必须插件化 | V4.0 插件 |
| 代码仓库集成 | 外部代码平台集成，必须插件化 | V4.0 插件 |
| AI | 模型 Provider 与推理能力，必须插件化 | V4.0 |
| 团队绩效 | 高级报表/绩效治理，不是 V1 Core | V4.0/V5.0 |
| 插件运行时/SDK/市场 | V1 只预留契约 | V4.0 |
| SSO/LDAP/多租户 | 企业商业能力 | V5.0 |
| 完整 Worklog/Time Tracking | Jira Core 能力，但不是 V1 基础闭环 | V3.0 |

> V1.0 可以保留任务上的“预估工时”字段。按天记录工时、剩余工时、工时统计、工时报表统一进入 V3.0，不作为 V1.0 发布阻塞。

## 4. 当前开发建议

从 2026-06-04 起，开发优先级如下：

| 顺序 | 现在要做 | 不要做 |
|------|----------|--------|
| 1 | 跑通后端编译、前端构建、登录和项目/任务 E2E | 不开发会议、AI、IM、WakaTime、CI/CD |
| 2 | 确认项目级权限和服务层权限兜底 | 不做 SSO/LDAP/多租户 |
| 3 | 完成任务详情、评论、关注、子任务、依赖 | 不做代码仓库深度集成 |
| 4 | 完成 Backlog/Sprint/Board 基础敏捷闭环 | 不做插件管理 UI |
| 5 | 完成站内信、邮件、附件、API Key、出站 Webhook | 不接飞书/钉钉/企微 SDK |
| 6 | 完成扩展点接口和事件命名文档 | 不做插件运行时和市场 |

插件相关 PRD/TASK 只保留 V4.0 入口；在 2027-09-06 前不进入开发冲刺。V5.0 企业身份、租户、合规和 SLA 能力按商业 Core 开发。

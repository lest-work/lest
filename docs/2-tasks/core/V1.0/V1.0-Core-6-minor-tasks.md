# Core V1.0 六个小版本任务总表

> **用途**：这是 V1.0 Core 的开发入口。开发同学可以直接按 1.1 到 1.6 执行，不需要重新通读全部 docs。
>
> **关联规划**：[V1.0 小版本规划](../../../MILESTONES/V1.0-小版本规划.md)
>
> **发布目标**：2026-08-28 发布 Jira-like Core 基础版。

## V1.0 范围冻结

| 进入 V1.0 Core | 不进入 V1.0 |
|----------------|-------------|
| 登录、刷新 Token、退出、个人资料、平台管理员 | SSO、LDAP、SCIM、多租户 |
| 项目 CRUD、归档、成员、项目角色、邀请 | OA 部门、岗位、菜单、字典管理 |
| 迭代、里程碑、Backlog、Sprint、Board | 会议、审批、OKR、Portfolio |
| Issue Key、固定 Issue 类型、固定状态、CRUD、详情 | 自定义 Issue Type、Workflow、Screen、Field Scheme |
| 评论、关注、标签、子任务、依赖、活动历史 | Worklog、批量操作、回收站、导入导出 |
| 站内信、WebSocket、SMTP 邮件、通知已读 | 飞书、钉钉、企业微信、Slack |
| 附件上传、下载、删除、元数据 | WakaTime、CI/CD、代码仓库 |
| 项目级权限、服务层兜底、基础审计 | AI、插件运行时、插件 SDK、插件市场 |
| API Key、出站 Webhook、OpenAPI | 第三方应用市场和沙箱 |

## 1.1 稳定性收口与 Core E2E

**时间**：2026-06-04 ~ 2026-06-20  
**目标**：系统能启动、能构建、能完成最小 Core 冒烟。

| 任务 ID | 任务 | 验收标准 | 优先级 |
|---------|------|----------|--------|
| V11-STAB-BE-001 | 后端核心服务编译修复 | gateway/auth/system/project/task/file/release 编译通过并可启动 | P0 |
| V11-STAB-FE-001 | 前端构建修复 | frontend-pc 构建通过，无缺失 store/style/API 阻塞 | P0 |
| V11-AUTH-001 | 登录/刷新/退出链路 | 登录、刷新 Token、退出、拉取个人信息可用 | P0 |
| V11-ADMIN-001 | 首个平台管理员初始化 | 首次启动可创建 platform_admin，且至少保留 1 名管理员 | P0 |
| V11-API-001 | API 路径核对 | 前端 API、Gateway、Controller 路径一致 | P0 |
| V11-SCHEMA-001 | Schema 核对 | DDL、Domain、Mapper、初始化数据一致 | P0 |
| V11-SEED-001 | 初始化数据 | 管理员、示例项目、基础状态、示例任务可初始化 | P0 |
| V11-OA-001 | OA 入口清理 | 前端不显示部门、岗位、菜单、字典等 OA 功能 | P0 |
| V11-E2E-001 | Core E2E 冒烟 | 登录 -> 建项目 -> 建任务 -> 拖看板 -> 评论 -> 通知通过 | P0 |

# 插件任务单

> **规划口径**：插件任务从 V4.0 开始正式进入开发排期。V1.0/V2.0/V3.0 只做 Core 和扩展契约，不保留插件任务入口。

## 快速导航

| 版本 | 定位 | TASK |
|------|------|------|
| V4.0 | 插件平台、官方插件池、AI 插件 | [V4.0](./V4.0/README.md) |
| V5.0 | 企业商业能力，不在插件目录重复维护 | [core/V5.0](../core/V5.0/README.md) |

## 插件开发原则

- Core First：先完成 V1.0~V3.0 Jira Core。
- 契约先行：V1.0~V3.0 只沉淀扩展点、事件、API 和 UI 插槽。
- 生态后置：完整插件 SDK、市场、热加载和官方插件开发从 V4.0 开始。
- 商业分层：SSO、LDAP、多租户、Portfolio、OKR、SLA 进入 V5.0 商业 Core。

## 1. 概述

### 1.1 功能背景

任务管理是 LEST Platform 的核心功能之一，为开发团队提供任务的创建、分配、跟踪和管理能力。任务可以归属到具体的迭代，也可以独立存在，支持看板视图、甘特图视图、列表视图等多种展示方式。

### 1.2 功能目标

- 提供任务的 CRUD 管理能力
- 支持多种视图展示（看板、甘特图、列表）
- 支持任务的分配、指派和认领
- 支持父子任务和任务依赖关系
- 支持工时记录和进度跟踪
- 与项目管理、代码管理深度集成

### 1.3 目标用户

- **项目经理**：创建任务、分配任务、跟踪进度
- **开发人员**：认领任务、更新进度、记录工时
- **产品经理**：创建需求类任务、跟踪需求实现进度
- **测试人员**：创建 Bug 任务、验证任务完成

## 2. 用户故事

### 2.1 任务基础管理

#### US-001: 创建任务
**作为** 项目经理，**我希望** 创建一个任务，**以便** 在项目中分配工作。

验收标准：
- [ ] 输入任务标题（必填）
- [ ] 选择所属项目（必填）和迭代（可选）
- [ ] 选择任务类型（需求/任务/Bug/优化）
- [ ] 选择优先级（紧急/高/中/低）
- [ ] 选择负责人（可选，不选则未分配）
- [ ] 设置预估工时（可选）
- [ ] 设置截止日期（可选）
- [ ] 富文本描述（可选，支持 Markdown）

#### US-002: 编辑任务
**作为** 任务负责人，**我希望** 编辑任务信息，**以便** 更新任务详情。

验收标准：
- [ ] 可编辑所有任务字段
- [ ] 编辑操作记录到操作日志
- [ ] 任务完成后不可再编辑（除非重新打开）

#### US-003: 删除任务
**作为** 项目管理员或任务创建者，**我希望** 删除任务，**以便** 清理不需要的任务。

验收标准：
- [ ] 删除前需确认
- [ ] 删除后子任务一并删除
- [ ] 删除后任务关联的评论、附件一并清理
- [ ] 删除操作记录到操作日志

#### US-004: 查询任务列表
**作为** 团队成员，**我希望** 查询任务列表，**以便** 找到需要处理的任务。

验收标准：
- [ ] 支持按项目筛选
- [ ] 支持按迭代筛选
- [ ] 支持按负责人筛选（查看我负责的任务）
- [ ] 支持按状态筛选
- [ ] 支持按类型筛选
- [ ] 支持按优先级筛选
- [ ] 支持按标签筛选
- [ ] 支持按关键字搜索标题和描述
- [ ] 支持分页，每页默认 20 条

#### US-005: 任务详情
**作为** 团队成员，**我希望** 查看任务详情，**以便** 了解任务的完整信息。

验收标准：
- [ ] 显示任务所有字段
- [ ] 显示任务关联的评论
- [ ] 显示任务关联的附件
- [ ] 显示任务关联的提交记录
- [ ] 显示任务的活动历史

### 2.2 任务分配与状态流转

#### US-006: 分配任务
**作为** 项目经理，**我希望** 将任务分配给团队成员，**以便** 明确责任人。

验收标准：
- [ ] 支持选择项目成员作为负责人
- [ ] 分配后发送通知给负责人
- [ ] 支持设置多个关注者，关注者会收到任务更新通知

#### US-007: 认领任务
**作为** 开发人员，**我希望** 主动认领未分配的任务，**以便** 主动承担工作。

验收标准：
- [ ] 未分配的任务显示"待认领"标签
- [ ] 任何项目成员可认领
- [ ] 认领后自动成为负责人

#### US-008: 状态流转
**作为** 任务负责人，**我希望** 更新任务状态，**以便** 反映任务进展。

验收标准：
- [ ] 状态流转：待办 → 进行中 → 已完成
- [ ] 状态变更为"进行中"时，自动设置开始时间
- [ ] 状态变更为"已完成"时，自动设置完成时间
- [ ] 只有负责人可以更新状态
- [ ] 管理员可以强制更新任意任务状态

#### US-009: 优先级处理
**作为** 项目经理，**我希望** 调整任务优先级，**以便** 优先处理重要任务。

验收标准：
- [ ] 支持四种优先级：紧急（P0）、高（P1）、中（P2）、低（P3）
- [ ] 紧急任务在最前面展示
- [ ] 支持批量调整优先级

### 2.3 看板视图

#### US-010: 看板展示
**作为** 团队成员，**我希望** 在看板视图中管理任务，**以便** 直观地了解项目进度。

验收标准：
- [ ] 默认列：待办、进行中、已完成
- [ ] 支持自定义列（通过项目设置配置）
- [ ] 每个列显示任务卡片数量
- [ ] 任务卡片显示：标题、负责人头像、优先级标签、截止日期
- [ ] 支持按负责人、优先级筛选看板内容

#### US-011: 拖拽任务
**作为** 团队成员，**我希望** 拖拽任务卡片到不同列，**以便** 快速更新任务状态。

验收标准：
- [ ] 拖拽到目标列后，任务状态自动变更
- [ ] 拖拽过程显示视觉反馈
- [ ] 拖拽后可撤销（Undo）

### 2.4 甘特图视图

#### US-012: 甘特图展示
**作为** 项目经理，**我希望** 在甘特图视图中查看任务时间线，**以便** 了解项目整体排期。

验收标准：
- [ ] 横轴为时间线（天/周/月）
- [ ] 纵轴为任务列表
- [ ] 每个任务显示为时间条
- [ ] 显示任务的时间跨度
- [ ] 显示任务负责人
- [ ] 支持缩放时间粒度

#### US-013: 甘特图交互
**作为** 项目经理，**我希望** 在甘特图中操作任务，**以便** 快速调整时间。

验收标准：
- [ ] 拖拽任务条调整开始/结束时间
- [ ] 拖拽任务条的边缘调整时长
- [ ] 点击任务条打开任务详情

### 2.5 父子任务与依赖

#### US-014: 创建子任务
**作为** 任务负责人，**我希望** 创建子任务，**以便** 拆分复杂任务。

验收标准：
- [ ] 一个任务最多支持 3 级子任务
- [ ] 子任务自动继承父任务的项目和迭代
- [ ] 父任务显示子任务数量和完成情况
- [ ] 删除父任务时可选是否删除子任务

#### US-015: 任务依赖
**作为** 任务负责人，**我希望** 设置任务依赖，**以便** 明确任务之间的先后顺序。

验收标准：
- [ ] 支持前置任务（Blocker）：当前任务等待前置任务完成后才能开始
- [ ] 支持阻塞任务（Blocked By）：显示当前任务被哪些任务阻塞
- [ ] 阻塞任务不可标记为"已完成"
- [ ] 支持依赖链的可视化展示

#### US-016: 父子任务进度
**作为** 项目经理，**我希望** 查看父子任务的进度，**以便** 了解子任务完成情况。

验收标准：
- [ ] 父任务自动计算子任务的完成百分比
- [ ] 父任务状态随子任务自动更新（所有子任务完成 → 父任务可完成）

### 2.6 工时管理

#### US-017: 记录工时
**作为** 任务负责人，**我希望** 记录任务的工作时长，**以便** 跟踪实际花费。

验收标准：
- [ ] 支持按天记录工时
- [ ] 输入工时（小时，精确到 0.5 小时）
- [ ] 输入工作描述（做了什么）
- [ ] 记录后自动累计到任务总工时

#### US-018: 工时统计
**作为** 项目经理，**我希望** 查看任务工时统计，**以便** 了解团队工作量分配。

验收标准：
- [ ] 显示任务的预估工时和实际工时
- [ ] 显示预估与实际的偏差
- [ ] 支持按成员统计工时
- [ ] 支持按项目统计工时
- [ ] 支持导出工时报表

### 2.7 标签管理

#### US-019: 管理任务标签
**作为** 项目管理员，**我希望** 管理项目下的任务标签，**以便** 按标签分类任务。

验收标准：
- [ ] 支持创建自定义标签（名称 + 颜色）
- [ ] 支持编辑和删除标签
- [ ] 标签颜色可选（预设颜色板）
- [ ] 一个任务可添加多个标签

#### US-020: 按标签筛选
**作为** 团队成员，**我希望** 按标签筛选任务，**以便** 快速找到特定类型的任务。

验收标准：
- [ ] 支持单选和多选标签筛选
- [ ] 筛选条件可保存为筛选器
- [ ] 常用筛选条件可收藏

### 2.8 任务-代码关联

#### US-021: 提交关联任务
**作为** 开发人员，**我希望** 将代码提交关联到任务，**以便** 追踪代码变更对应的任务。

验收标准：
- [ ] 支持通过 Commit Message 关键字自动关联（如 `Closes #123`、`Fix #123`、`[task-123]`）
- [ ] 支持手动关联提交记录
- [ ] 任务详情显示关联的 Commit 列表（SHA、message、author、date）
- [ ] 支持关联 MR/PR（源分支、目标分支、状态、链接）

#### US-022: 查看任务关联的代码变更
**作为** 项目经理，**我希望** 查看任务关联的代码变更，**以便** 了解任务的实现进度。

验收标准：
- [ ] 任务详情显示关联的 Commit 列表
- [ ] 任务详情显示关联的 MR/PR 列表
- [ ] 支持从 Commit 跳转到代码仓库
- [ ] 支持从 MR 跳转到代码仓库

### 2.9 任务-CI 关联

#### US-023: 查看任务关联的构建结果
**作为** 开发人员，**我希望** 在任务详情中查看关联的 CI 构建结果，**以便** 了解代码部署状态。

验收标准：
- [ ] 任务详情显示关联的 Pipeline/Build 列表
- [ ] 显示构建状态（成功/失败/运行中）
- [ ] 支持从任务跳转查看构建详情
- [ ] 支持从构建结果反查关联的任务

#### US-024: 构建失败自动创建 Bug 任务
**作为** 运维人员，**我希望** 配置 CI 构建失败时自动创建 Bug 任务，**以便** 跟踪构建问题。

验收标准：
- [ ] 支持配置自动创建 Bug 任务的触发条件（仅失败/失败+N次）
- [ ] 自动创建的 Bug 任务关联对应的构建记录
- [ ] 自动填充 Bug 任务的标题和描述（包含构建信息）
- [ ] 支持配置自动创建的任务类型、优先级、负责人

### 2.10 任务-通知规则

#### US-025: 接收任务变更通知
**作为** 任务负责人或关注者，**我希望** 接收任务变更通知，**以便** 及时了解任务动态。

验收标准：
- [ ] 任务被分配时通知负责人
- [ ] 任务状态变更时通知负责人和关注者
- [ ] 任务被评论时通知被 @提及的用户
- [ ] 任务截止日期临近时（1天/3天）通知负责人
- [ ] 任务逾期时通知负责人和任务创建者
- [ ] 任务依赖被解除时通知负责人

### 2.11 跨项目任务依赖

#### US-026: 设置跨项目任务依赖
**作为** 项目经理，**我希望** 设置跨项目任务依赖，**以便** 协调多团队协作。

验收标准：
- [ ] 支持选择其他项目的任务作为前置任务
- [ ] 跨项目依赖任务显示为"外部任务"，仅显示标题和状态
- [ ] 跨项目依赖变更时，双方项目成员均收到通知
- [ ] 查询跨项目依赖时，需校验用户对目标项目的访问权限
- [ ] 循环依赖检测需覆盖跨项目场景

### 3.3 任务类型

### 3.6 任务通知事件定义

| 事件编码 | 事件名称 | 触发时机 | 通知渠道 | 接收者 | 优先级 |
|---------|---------|---------|---------|-------|--------|
| TASK_CREATED | 任务创建 | 创建新任务时 | 站内信、邮件 | 负责人（如有） | P0 |
| TASK_ASSIGNED | 任务分配 | 分配负责人时 | 站内信、邮件 | 被分配的负责人 | P0 |
| TASK_STATUS_CHANGED | 任务状态变更 | 状态变更时 | 站内信 | 负责人、关注者 | P0 |
| TASK_PRIORITY_CHANGED | 任务优先级变更 | 优先级调整时 | 站内信 | 负责人 | P1 |
| TASK_COMMENTED | 任务评论 | 发表评论时 | 站内信 | 被@提及者、任务负责人 | P0 |
| TASK_DUE_SOON | 任务即将到期 | 截止日期前 1/3 天 | 站内信、邮件 | 负责人 | P1 |
| TASK_OVERDUE | 任务已逾期 | 超过截止日期 | 站内信、邮件 | 负责人、任务创建者 | P0 |
| TASK_COMPLETED | 任务完成 | 状态变为已完成时 | 站内信 | 任务创建者 | P1 |
| TASK_DEPENDENCY_REMOVED | 依赖解除 | 前置任务完成解除阻塞时 | 站内信 | 任务负责人 | P1 |
| TASK_LINKED_TO_MR | 任务关联 MR | MR 创建或关联任务时 | 站内信 | 任务负责人 | P2 |
| TASK_LINKED_TO_COMMIT | 任务关联提交 | 提交关联到任务时 | 站内信 | 任务负责人 | P2 |

### 3.7 跨项目任务依赖规则

| 规则 | 说明 |
|------|------|
| 依赖可见性 | 跨项目依赖时，外部任务仅显示标题和状态，不暴露项目内部信息 |
| 权限校验 | 查询跨项目依赖时，需校验用户对目标项目有访问权限 |
| 循环依赖检测 | 跨项目场景下仍需检测循环依赖，禁止跨项目循环 |
| 变更通知 | 跨项目依赖状态变更时，双方项目成员均收到通知 |
| 依赖移除 | 移除跨项目依赖时，仅影响当前项目，己方任务不受影响 |

  "data": {
    "autoLinkedTaskIds": [1]
  }
}

说明: 自动解析 commit_message 中的任务 ID (Closes #N / Fix #N / [task-N])
并创建 task_commit 关联记录

## 8. 错误码

### 8.1 任务模块错误码 (8000-8999)

| 错误码 | 枚举常量 | HTTP 状态码 | 说明 |
|---------|----------|-------------|------|
| 8000 | `TASK_NOT_FOUND` | 404 | 任务不存在 |
| 8001 | `TASK_DELETED` | 400 | 任务已删除 |
| 8002 | `TASK_COMPLETED` | 400 | 任务已完成，无法编辑 |
| 8003 | `TASK_HAS_DEPENDENCIES` | 400 | 任务存在依赖关系，无法完成 |
| 8004 | `TASK_BLOCKED_BY_OTHERS` | 400 | 任务被其他任务阻塞，无法完成 |
| 8005 | `TASK_PARENT_NOT_FOUND` | 404 | 父任务不存在 |
| 8006 | `TASK_DEPTH_EXCEEDED` | 400 | 子任务层级超过 3 级 |
| 8007 | `TASK_CIRCULAR_DEPENDENCY` | 400 | 存在循环依赖 |
| 8008 | `TASK_ASSIGNEE_NOT_PROJECT_MEMBER` | 400 | 指派的用户不是项目成员 |
| 8009 | `TASK_COMMIT_NOT_FOUND` | 404 | 提交关联记录不存在 |
| 8010 | `TASK_MR_NOT_FOUND` | 404 | MR 关联记录不存在 |
| 8011 | `TASK_PIPELINE_NOT_FOUND` | 404 | 构建关联记录不存在 |
| 8012 | `TASK_CROSS_PROJECT_ACCESS_DENIED` | 403 | 无权访问跨项目任务依赖 |
| 8013 | `TASK_CROSS_PROJECT_CIRCULAR` | 400 | 跨项目循环依赖 |
| 8014 | `TASK_LINK_DUPLICATE` | 409 | 关联记录已存在 |

### 8.2 工时模块错误码 (8100-8199)

| 错误码 | 枚举常量 | HTTP 状态码 | 说明 |

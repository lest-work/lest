## 1. 概述

### 1.1 功能背景

LEST Platform 作为开源的 AI-Native 敏捷管理平台，需要提供强大的扩展能力，允许开发者创建、分享和安装插件来扩展平台功能。这借鉴了 VS Code、Jenkins、WordPress 等成熟平台的插件生态模式。

**核心设计理念**：
- **可插拔**：平台核心保持精简，业务功能通过插件扩展
- **可发现**：插件市场提供发现和分发能力
- **安全隔离**：插件运行在隔离环境中，不影响主系统稳定性
- **版本兼容**：通过语义化版本确保插件与平台版本的兼容性

### 1.2 功能目标

- 提供统一的插件开发框架（SDK），支持 Java/TypeScript 双语言
- 提供插件生命周期管理（安装、启用、禁用、卸载、升级）
- 提供声明式扩展点机制，允许插件在指定位置注入功能
- 提供插件市场，支持插件的发现、安装、评价
- 提供插件沙箱隔离，确保系统安全稳定
- 支持插件数据隔离，每个插件有独立的数据空间

### 1.3 与开放平台的关系

| 模块 | 职责 | 关系 |
|------|------|------|
| 插件系统 | 提供运行时扩展能力 | 扩展平台功能 |
| 开放平台 | 提供外部集成能力 | 扩展平台连接性 |
| AI 服务 | 提供 AI 能力扩展点 | 插件的一种类型 |

插件系统和开放平台是 LEST Platform 扩展生态的两大支柱，共同支撑平台的开放性。

## 2. 用户故事

### 2.1 插件开发

#### US-001: 创建插件项目
**作为** 开发者，**我希望** 使用 CLI 工具创建插件项目，**以便** 快速开始插件开发。

验收标准：
- [ ] `lest plugin init` 命令创建标准插件项目结构
- [ ] 自动生成插件元数据文件（plugin.json）
- [ ] 提供 TypeScript 和 Java 两种语言模板
- [ ] 支持在 IDE 中进行调试

#### US-002: 定义扩展点
**作为** 开发者，**我希望** 在插件中声明扩展点，**以便** 在平台特定位置注入功能。

验收标准：
- [ ] 支持声明式扩展点定义（plugin.json）
- [ ] 支持多种扩展点类型：UI 组件、API 路由、菜单项、Webhook 处理器、Scheduled Job
- [ ] 支持扩展点参数定义和验证

#### US-003: 调用平台 API
**作为** 开发者，**我希望** 在插件中调用平台提供的 API，**以便** 读取和操作业务数据。

验收标准：
- [ ] 提供 Platform API Client SDK
- [ ] 支持读取/创建/更新/删除任务、项目、用户等数据
- [ ] 支持权限校验和审计日志
- [ ] 支持 API 版本管理

#### US-004: 打包和发布插件
**作为** 开发者，**我希望** 打包插件并发布到插件市场，**以便** 让其他用户使用我的插件。

验收标准：
- [ ] `lest plugin build` 命令生成插件包（.lpkg 文件）
- [ ] 支持插件签名验证
- [ ] 支持发布到官方插件市场或私有仓库
- [ ] 支持版本管理和更新推送

    private String name;             // 扩展点名称

}

### 3.4 插件隔离机制

#### 3.4.1 类加载器隔离

每个插件使用独立的 ClassLoader，防止类冲突：
- 插件依赖与平台依赖隔离
- 插件之间相互隔离
- 支持同一依赖的不同版本共存

#### 3.4.2 数据库隔离

每个插件拥有独立的数据表前缀：
```
插件 A: plugin_a_task, plugin_a_config, plugin_a_log
插件 B: plugin_b_issue, plugin_b_setting
平台核心: task, project, user
```

#### 3.4.3 API 隔离

- 插件自定义 API 路径统一前缀：`/api/plugin/{pluginId}/*`
- 插件 API 共享平台的认证和权限机制
- 支持插件级别的限流配置

import { PlatformClient } from '@lest/sdk-client';

    // 事件发布

## 4. 接口设计

### 4.1 插件管理接口

#### 4.1.1 获取已安装插件列表
```
GET /plugin/list
Authorization: Bearer {accessToken}

响应:
{
  "code": 200,
  "message": "success",
  "data": [
    {
      "id": "lest-issue-link",
      "name": "Jira 问题同步",
      "version": "1.0.0",
      "status": "enabled",
      "author": "LEST Team",
      "enabledAt": "2026-05-20 10:00:00",
      "lastUpdated": "2026-05-20 10:00:00"
    }
  ]
}
```

#### 4.1.2 安装插件
```
POST /plugin/install
Authorization: Bearer {accessToken}
Content-Type: application/json

请求:
{
  "source": "market",     // market | file | url
  "pluginId": "lest-issue-link",  // source=market 时必填
  "version": "1.0.0",    // 可选，默认安装最新
  "file": "...",         // source=file 时，base64 编码的文件内容
  "url": "..."           // source=url 时，插件包下载地址
}

响应:
{
  "code": 200,
  "message": "success",
  "data": {
    "id": "lest-issue-link",
    "name": "Jira 问题同步",
    "version": "1.0.0",
    "status": "enabled"
  }
}
```

#### 4.1.3 启用/禁用插件
```
POST /plugin/{pluginId}/enable
POST /plugin/{pluginId}/disable
Authorization: Bearer {accessToken}

响应:
{
  "code": 200,
  "message": "success"
}
```

#### 4.1.4 卸载插件
```
DELETE /plugin/{pluginId}?keepData=true
Authorization: Bearer {accessToken}

参数:
- keepData: boolean, 是否保留插件数据, 默认 false

响应:
{
  "code": 200,
  "message": "success"
}
```

#### 4.1.5 升级插件
```
POST /plugin/{pluginId}/upgrade
Authorization: Bearer {accessToken}
Content-Type: application/json

请求:
{
  "targetVersion": "1.1.0"
}

响应:
{
  "code": 200,
  "message": "success",
  "data": {
    "id": "lest-issue-link",
    "oldVersion": "1.0.0",
    "newVersion": "1.1.0"
  }
}
```

## 1. 概述

### 1.1 功能背景

代码管理是 LEST Platform 连接代码仓库与项目管理的重要桥梁。通过深度集成 GitHub 和 GitLab，团队可以在平台上统一管理代码仓库、追踪 Pull Request/Merge Request、关联提交与任务，实现代码级别的项目追踪。

### 1.2 功能目标

- 支持 GitHub 和 GitLab 双平台集成
- 提供仓库的统一管理视图
- 支持 MR/PR 的创建、审查、合并流程
- 实现提交与任务的关联追踪
- 展示 CI/CD 构建状态
- 为 AI 代码审查提供数据基础

### 1.3 目标用户

- **开发人员**：提交代码、创建 MR/PR、参与代码审查
- **项目经理**：查看代码提交情况、MR 进度
- **代码审查者**：在平台内进行代码审查
- **平台管理员**：管理仓库关联配置

### 4.4 Git Webhook 接口

#### 4.4.1 GitHub Webhook 回调
```
POST /webhook/github
X-Hub-Signature-256: {签名}
X-GitHub-Event: {事件类型}

Body: (GitHub Webhook Payload)
```

#### 4.4.2 GitLab Webhook 回调
```
POST /webhook/gitlab
X-Gitlab-Token: {签名}

Body: (GitLab Webhook Payload)
```

#### 4.4.3 Webhook 事件处理矩阵

| 事件类型 | 来源 | 处理逻辑 | 触发的跨模块事件 |
|---------|------|---------|---------------|
| push | GitHub/GitLab | 解析 commit_message，自动关联任务 | 向任务模块发送 `/webhook/git/commit` |
| pull_request (opened) | GitHub | 解析任务 ID，自动关联 | 向任务模块发送 `TASK_LINKED_TO_MR` |
| pull_request (merged) | GitHub | 更新 MR 状态 | 向任务模块发送 `MR_MERGED` |
| pull_request (closed) | GitHub | 更新 MR 状态 | 向任务模块发送 `MR_CLOSED` |
| merge_request (opened) | GitLab | 解析任务 ID，自动关联 | 向任务模块发送 `TASK_LINKED_TO_MR` |
| merge_request (merged) | GitLab | 更新 MR 状态 | 向任务模块发送 `MR_MERGED` |
| note (commit comment) | GitHub/GitLab | 解析任务 ID，发送通知 | 向通知模块发送 `TASK_COMMENTED` |

### 4.6 代码模块通知事件定义

| 事件编码 | 事件名称 | 触发时机 | 通知对象 | 说明 |
|---------|---------|---------|---------|------|
| MR_CREATED | MR 创建 | 创建 MR 时 | 被指定的审查者 | US-005 |
| MR_REVIEW_REQUESTED | 请求审查 | 指定审查者时 | 被请求的审查者 | US-005 |
| MR_REVIEWED | 审查完成 | 审查者提交审查意见时 | MR 作者 | US-006 |
| MR_COMMENTED | MR 评论 | 发表评论时 | 被@提及者、MR 作者 | US-006 |
| MR_MERGED | MR 合并 | 代码合并时 | MR 作者、相关开发者 | US-007 |
| MR_CLOSED | MR 关闭 | MR 被关闭时 | MR 作者 | US-007 |

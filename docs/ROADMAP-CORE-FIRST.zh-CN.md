# LEST Platform Core-first 路线图

> **更新时间**：2026-06-04
>
> **定位**：先完成 Jira Core 的真实可用闭环，再开放插件生态、AI 原生能力和商业企业能力。
>
> **原则**：V1.0 不交付官方插件功能，只交付 Core 能力和扩展矩阵契约。插件 PRD 作为后续插件池，不阻塞 V1.0。
>
> **开发执行入口**：[DEVELOPMENT-EXECUTION-PLAN.zh-CN.md](./DEVELOPMENT-EXECUTION-PLAN.zh-CN.md) — 开发排期优先看这份，不需要重新通读全部 docs。

---

## 1. 产品定位

LEST Platform 的第一目标不是一次性复制 Jira 的所有周边生态，而是先做出一个可私有化部署、可持续扩展、核心流程稳定的 Jira-like 敏捷管理平台。

### 1.1 核心承诺

| 方向 | 承诺 |
|------|------|
| Jira Core | 项目、任务、看板、Backlog、Sprint、通知、权限、附件、工时、报表、自动化逐步完整 |
| Scheme 架构 | Issue Type / Workflow / Screen / Field / Permission / Notification 可配置 |
| 扩展矩阵 | Core 内置扩展点和契约，第三方能力通过插件接入 |
| 中国企业适配 | IM、审批、组织同步、国产化作为 V4.0 插件或 V5.0 企业能力 |
| 商业化 | 开源 Core 保持可用，企业版提供多租户、SSO、LDAP、审计合规、SLA、治理能力 |

### 1.2 不做的事

| 阶段 | 明确不做 |
|------|----------|
| V1.0 | 不开发会议、CI/CD、WakaTime、绩效、AI、IM 等官方插件 |
| V1.0 | 不做插件市场、热部署、第三方沙箱、完整 SDK |
| V2.0 | 不追求 Jira 全量边角功能，优先完成 Scheme 主链路 |
| V3.0 | 不开发会议、CI/CD、WakaTime、AI、IM、OKR、Portfolio、审批、SSO/LDAP、多租户 |

## 2. 版本总览

| 版本 | 定位 | 核心目标 | 许可 |
|------|------|----------|------|

## 3. 0.x 预发布规划

### 0.1 — 可运行骨架

目标是让系统能稳定启动和演示，不追求功能完整。

| 优先级 | 交付项 | 验收标准 |
|--------|--------|----------|
| P0 | 后端编译通过 | gateway/auth/system/project/task/file/release 可启动 |
| P0 | 前端构建通过 | frontend-pc 无缺失 store/style/API 导致的构建错误 |
| P0 | 登录链路 | 登录、刷新 Token、退出、个人信息可用 |
| P0 | 项目列表 | 创建、查看、归档项目可用 |
| P0 | 任务列表/看板 | 创建任务、筛选、看板拖拽可用 |
| P1 | 删除 OA 页面入口 | 用户看不到部门、岗位、菜单、字典等 OA 功能 |

### 0.2 — Core 闭环补齐

目标是把最小 Jira Core 流程跑通。

| 优先级 | 交付项 | 验收标准 |
|--------|--------|----------|
| P0 | 项目成员与角色 | Admin/Member/Viewer 生效 |
| P0 | 项目级权限校验 | 非成员不能读写项目任务 |
| P0 | 任务详情 | 标题、描述、负责人、状态、优先级、评论、关注可编辑 |
| P0 | 子任务/依赖 | 创建、展示、阻塞校验可用 |
| P0 | 通知中心 | 指派、评论、@ 提及、状态变更产生站内通知 |
| P1 | 迭代/里程碑 UI | 项目内 Sprint 与里程碑可维护 |
| P1 | 附件基础能力 | 任务附件上传、下载、删除 |

### 0.3 — 发版候选

目标是把 V1.0 的风险提前清掉。

| 优先级 | 交付项 | 验收标准 |
|--------|--------|----------|
| P0 | API 路径对齐 | 前端 API 与后端 Controller/Gateway 全部一致 |
| P0 | Schema 对齐 | Domain、Mapper、DDL、初始化数据一致 |
| P0 | 审计基础 | 核心写操作记录 actor/action/resource/time |
| P0 | E2E 冒烟 | 登录 -> 建项目 -> 加成员 -> 建任务 -> 拖看板 -> 评论 -> 通知 |
| P1 | Docker Compose | 本地一键启动核心依赖和服务 |
| P1 | OpenAPI 文档 | Core API 可浏览、可调试 |

## 4. V1.0 — Jira Core 基础版

> **目标**：让小团队可以用 LEST 管项目、管任务、跑看板和 Sprint。V1.0 只做 Core，不开发官方插件。

### 4.1 V1.0 范围

| 模块 | 必须交付 |
|------|----------|
| 认证与用户 | 登录、Token、用户资料、密码、平台管理员 |
| 项目管理 | 项目 CRUD、归档、成员、角色、迭代、里程碑 |
| 任务管理 | Issue CRUD、评论、关注、子任务、依赖、基础历史 |
| 看板 | To Do / In Progress / Done、拖拽、基础筛选 |
| Backlog/Sprint | Backlog、Sprint 开始/完成、任务移入移出 |
| 通知 | 站内信、WebSocket、@ 提及、通知已读 |
| 附件 | 上传、下载、删除、大小限制 |
| 权限 | 项目级角色校验、平台管理员绕过、基础审计 |
| 系统设置 | 邮件配置、安全设置、审计日志查看 |
| 扩展矩阵 | 后端扩展点接口定义、前端扩展槽位预留、事件命名规范 |
| 开放 API | API Key、Webhook 出站、基础限流 |

### 4.2 V1.0 明确延期

| 功能 | 延期到 |
|------|--------|
| 会议管理 | V4.0 官方插件池 |
| CI/CD 集成 | V4.0 官方插件池 |
| WakaTime/编码追踪 | V4.0 官方插件池 |
| 团队绩效 | V4.0 官方插件池或 V5.0 企业报表 |
| AI 助手 | V4.0 |
| 企业微信/钉钉/飞书深度集成 | V4.0 插件，企业组织同步可进入 V5.0 |
| 插件市场/SDK/热加载 | V4.0 |

### 4.3 V1.0 不可发布条件

- 后端核心服务无法编译或启动
- 前端无法构建
- 项目级权限缺失
- 核心 E2E 冒烟不通过
- API 文档与真实接口不一致
- 任务详情、看板、通知任一核心链路不可用

## 5. V2.0 — Scheme 可配置版

> **目标**：把 LEST 从固定任务系统升级为 Jira 式可配置平台。

### 5.1 推荐实现顺序

| 小版本 | 主题 | 交付 |
|--------|------|------|
| 2.1 | Issue Type | Epic/Story/Task/Bug/Subtask、任务编号、类型方案 |
| 2.2 | Workflow | 状态、流转、条件、校验、后置动作、工作流方案 |
| 2.3 | Custom Field | 先支持 8 类高频字段，再扩展到 25 类 |
| 2.4 | Screen / Field Config | 创建/编辑/查看屏幕、字段必填/隐藏/只读 |
| 2.5 | Project Template | 敏捷、Bug 跟踪、看板、空白项目模板 |
| 2.6 | JQL Lite + 迁移 | 保存过滤器、基础 JQL、V1 数据迁移、性能优化 |

### 5.2 字段类型分期

| 阶段 | 字段类型 |
|------|----------|
| V2.3 P0 | 文本、长文本、数字、日期、单选、多选、用户、URL |
| V2.3 P1 | 日期时间、复选框、级联选择、版本、组件、标签 |
| V3.0 | 公式、只读派生、远程数据、复杂搜索器 |

## 6. V3.0 — Jira Core 完整版

> **目标**：完成 Jira Core 的关键缺口，不把中国企业协作插件作为 Core 发布阻塞项。

| 小版本 | 主题 | 交付 |
|--------|------|------|
| 3.1 | Issue Link + Attachment | 关联、附件版本、预览、权限 |
| 3.2 | Time Tracking + Worklog | 预估、剩余、已记录、工时日志、汇总 |
| 3.3 | Automation | 规则引擎、执行日志、模板、预览 |
| 3.4 | Agile Reports | 燃尽图、CFD、Velocity、Control Chart、Epic 进度 |
| 3.5 | Bulk + Shortcut | 批量编辑/移动/删除/流转、快捷键 |
| 3.6 | Search + Filter | JQL 完整化、保存过滤器、订阅、导入导出 |
| 3.7 | Jira Import | Jira/禅道导入、字段映射、迁移报告 |

## 7. V4.0 — 扩展生态与 AI

> **目标**：在 Core 稳定后开放插件生态，并把 AI 作为可插拔能力接入。

### 7.1 插件平台

| 能力 | 交付 |
|------|------|
| 插件生命周期 | 安装、启用、禁用、配置、卸载 |
| 插件 SDK | 后端 SDK、前端 SDK、事件订阅、API Client |
| 前端扩展点 | Dashboard Widget、Task Detail Tab、Project Tab、Board Action、Command Palette |
| 后端扩展点 | Workflow Action、Automation Action、Notification Channel、Import/Export Provider |
| 官方插件池 | IM、CI/CD、代码仓库、会议、WakaTime、报表、Jira Sync |

### 7.2 AI 能力

| 能力 | 许可建议 |
|------|----------|
| AI 任务拆解 | 开源 |
| AI 自然语言查询 | 开源 |
| AI 风险识别 | 开源 |
| AI 代码质量分析 | 开源或商业增强 |
| AI 智能排期 | 商业 |
| AI 报告/文档生成 | 商业 |

## 8. V5.0 — 企业商业版

> **目标**：面向中大型企业、金融政务、集团型组织。

| 能力 | 商业版范围 |
|------|------------|
| 多租户 | 租户隔离、租户配置、租户管理员 |
| 身份集成 | LDAP、OIDC、SAML SSO、SCIM 用户同步 |
| 审计合规 | 审计检索、导出、留存策略、SIEM 集成 |
| 精细权限 | Permission Scheme、Issue Security、字段级屏蔽 |
| 企业报表 | 跨项目报表、Portfolio、OKR、资源容量 |
| 高可用 | K8s、Helm、备份恢复、灰度发布 |
| SLA | 官方支持、升级服务、故障响应 |

## 9. 文档维护规则

1. `docs/ROADMAP-CORE-FIRST.zh-CN.md` 是路线图主文档。
2. `docs/MILESTONES.zh-CN.md` 只保留版本摘要和链接。
3. `docs/MILESTONES/V*.md` 只描述该大版本交付，不跨版本塞功能。
4. 插件 PRD/TASK 只保留 V4.0 入口；V1.0/V2.0/V3.0 不保留插件目录，V5.0 企业身份、租户、合规和 SLA 能力归入商业 Core。
5. 所有版本必须区分“PRD 已设计”和“代码已交付”。
